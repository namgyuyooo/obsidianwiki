수집 후 위키화 단순 실행

사용자는 아래 순서만 보면 됩니다. 원본 Google Drive는 절대 삭제하지 않고, hwp/hwpx/pdf/docx/pptx/html 보고서 등은 로컬 mirror에 복사한 뒤 위키화합니다.

기본 경로: gdrive: 최상위
1 Drive 수집 미리보기

보수적인 rclone 제한값으로 어떤 파일을 가져올지 먼저 확인합니다.

2 Drive 수집 실행

원본 Drive는 건드리지 않고 rclone copy로 로컬 mirror에만 내려받습니다.

3 Manifest 생성

로컬 mirror에 내려온 hwp/hwpx/pdf/docx/pptx/html 등 문서 목록을 만듭니다.

4 위키화 실행

청크 요약, 파일 요약, 프로젝트 분기, 위키 반영을 작은 배치로 실행합니다.

5 전체 흐름 미리보기

copy, manifest, run 흐름을 안전하게 dry-run으로 확인합니다.

6 오픈클로/GLM 트리거

OpenClaw webhook이 없으면 로컬 Drive Wikify dry-run을 실제 실행합니다.

! 중지

실행 중인 로컬 파이프라인만 중지합니다. 원본 Drive 삭제 명령은 없습니다.

현재 실행 상태

대기 중

권장 순서: 표적 분석 -> 필요한 경로만 rclone copy -> manifest 생성 -> 위키화 실행. 하루 종일 돌릴 때도 이 로그 기준으로 재개합니다.

지시 대기
지시문을 넣으면 키워드, 후보 경로, 안전 실행 계획이 표시됩니다.
