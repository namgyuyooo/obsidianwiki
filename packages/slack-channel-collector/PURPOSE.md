# Slack Channel Collector Purpose

이 패키지는 RTM Slack의 `#tf_cross_team_sales` 채널을 하루 한 번 수집해 JSON 증거 파일로 보존하기 위한 독립 모듈입니다.

## 왜 분리했나

기존 위키 자동화 수집기는 Slack raw export 이후 업무 메시지 필터링, 위키 문서 append, 프로젝트 공간 생성까지 한 번에 수행합니다. 정기 수집만 필요한 환경에서는 그 범위가 너무 넓습니다.

이 패키지는 다음 원칙으로 경계를 좁힙니다.

- Slack은 읽기 전용으로만 접근한다.
- 결과는 JSON 파일과 state 파일로만 남긴다.
- 위키 구조, 프로젝트 판단, LLM 필터는 호출하지 않는다.
- `#tf_cross_team_sales` 운영 채널을 기본값으로 고정하되, 다른 채널명/ID도 옵션으로 받을 수 있다.

## 무엇을 포함하나

- Slack API token 기반 채널 조회
- 채널 히스토리 수집
- 스레드 답글 수집 옵션
- 파일 메타데이터 보존 옵션
- KST 기준 매일 24:00 실행 루프
- 증분 수집 state
- 패키지 내 기본 env 파일
- 기존 솔루션의 Slack/GLM 계열 환경 키 호환

## 무엇을 제외하나

- 위키 승격
- 요약/분류/LLM 필터링
- 첨부파일 다운로드
- 문서 텍스트 추출
- Slack write API 사용

## 권장 역할

이 모듈은 downstream 분석기, 위키화 파이프라인, BI 적재 작업 앞단에 놓는 원천 JSON 수집기로 쓰는 것이 좋습니다. 이후 정리/요약/위키 승격은 별도 모듈이 JSON을 읽어 수행해야 합니다.
