# 현대모비스 — L1 Memory Snapshot

## 한줄 요약
ICT AI 솔루션 개발 요구사항·제작사양 구체화 완료. 하지만 납기-공수 간극이 핵심 리스크. 견적서 수정 요청 발생 상태.

## 프로젝트 유형
요구사항 구체화 → 계약 전환 단계 / 납기 리스크 관리 중요

## 현재 상태 (What's happening now)
- 요구사항 확인 문서 + 제작사양서 존재 (2026-03 말 기준)
- 견적서 수정 요청 발생 (본사 심의 대응용)
- 계약서상 납기 목표 vs 내부 실제 수행 가능 기간 충돌 중

## 핵심 결정사항
- 요구사항/제작사양 → 구현 수준 논의 시작 단계
- 견적서는 심의 대응 기준으로 수정 필요

## 핵심 수치 / 파일
- 문서: 요구사항 확인 문서, 제작사양서, 견적서
- 쟁점: 납기 목표 vs 실무 공수 간극

## 미해결 이슈 / 확인 필요
- 최종 견적서 수정안이 무엇으로 정리됐는지
- 실제 계약안에서 납기가 어떻게 반영됐는지

## 주의사항 (Gotchas)
- "무엇을 만드나"보다 "언제까지 가능한가"가 핵심 리스크
- 요구사항과 제작사양서를 같이 봐야 범위 파악 가능 — 둘 중 하나만 보면 안됨
- 본사 심의 절차가 있으므로 견적 방어 논리 사전 준비 필요
