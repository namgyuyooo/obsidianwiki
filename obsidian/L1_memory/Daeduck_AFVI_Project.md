# 대덕전자 AFVI — L1 Memory Snapshot

## 한줄 요약
PCB/패키지 AFVI 검사 제안 프로젝트. ODB 데이터·크레셈 연동·SDK 개발 방향까지 논의된 제안-기술검토형. 임원진 불안이 내부 리스크.

## 프로젝트 유형
제안-기술검토형 / 과제화 직전 또는 초입 단계

## 현재 상태 (What's happening now)
- 제안서 3개 버전 존재 (231004 → 231128 → 231218 ODB 추가)
- 상세 업무·공수·일정 보강 필요하다는 내부 판단 확인됨
- 고객/임원 관점에서 AI 진행 가시성 부족 우려 남아 있음

## 핵심 결정사항
- AFVI는 ODB 데이터 + 설비/시스템 인터페이스 정의까지 포함하는 방향
- ODB 원본은 고객 자산 → RTM은 고객 제공 범위에서만 분석
- 기존 제안서는 공수/일정 보강 필요 결론

## 핵심 수치 / 파일
- 제안서: `01. 대덕전자 AFVI 제안서_231004.pptx`, `_231128.pptx`, `231218_ODB내용추가.pptx`
- 기술: `DD_afvi_도면여부_flow.pptx`, feature 파일들, 네트워크 구성도
- 협력사: 크레셈 (SDK 개발, I/O 정의, 데이터 전송 방식, 사용 Language 확인 필요)

## 미해결 이슈 / 확인 필요
- AFVI 1차 제안과 ODB 추가 버전이 연속 과제인지
- 크레셈 연동 요구가 실제 수주 범위에 포함됐는지
- 임원진 불안 이후 고객 보고 방식·일정 어떻게 조정됐는지

## 주의사항 (Gotchas)
- 기술 검토가 깊어질수록 오히려 진행 가시성이 떨어지는 패턴 — 고객 커뮤니케이션 별도 관리 필요
- FCCSP 특화 도메인, 데이터 소유권 이슈 사전 정리 없으면 제안 막힘
