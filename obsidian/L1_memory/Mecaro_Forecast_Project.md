# 메카로 수요예측 — L1 Memory Snapshot

## 한줄 요약
Pre-PoC → PoC 결과발표 → 단계별 제안서 → 근거자료 보강 → 생산/설비팀 미팅까지 진행. 모델 성능보다 제안 구조와 FCST 근거자료 정합성이 핵심.

## 프로젝트 유형
제안-전환형 / 현업 적용 검토 시작 단계

## 현재 상태 (What's happening now)
- PoC 결과발표 완료, 단계별 제안서 존재
- FCST 근거자료 문의가 별도 발생 → 제안 방어 논리 보강 중
- 생산/설비팀 미팅까지 이어져 현업 적용 가능성 검토 시작

## 핵심 결정사항
- PoC 결과를 사업 제안 구조로 번역하는 단계까지 진행
- 판매계획 데이터 자산화가 핵심 가치 제안

## 핵심 수치 / 파일
- 문서: Pre-PoC 리뷰, PoC 결과발표, 단계별 제안서, FCST 근거자료 문의, 생산/설비팀 미팅
- 주제: FCST, 수요예측, 판매계획 데이터 자산화

## 미해결 이슈 / 확인 필요
- PoC 결과발표 버전별 차이
- 근거자료 문의에 대한 후속 답변과 실제 반영 여부

## 주의사항 (Gotchas)
- 모델 정확도보다 "왜 이 예측을 믿어야 하나"라는 근거 방어가 핵심
- PoC 결과발표와 단계별 제안서를 같이 봐야 사업화 논리 파악 가능
- 수요예측 계열 제안의 템플릿 사례로 재사용 가치 높음
