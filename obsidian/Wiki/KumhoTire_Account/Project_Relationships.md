# Project Relationships

금호타이어 계정은 현재 위키상 두 개의 주요 workstream으로 나뉜다. 하나는 `첫제품 X-ray` 축이고, 다른 하나는 `CMS` 축이다. 두 프로젝트는 고객은 같지만, 증적의 성격과 현장 문제 정의가 다르기 때문에 하위 프로젝트를 분리해 관리하는 편이 적절하다.

## FirstProduct X-ray

- 중간보고, 사용자 매뉴얼, 산출물 목록, 현장회의, 설치 전환 흐름이 중심이다.
- `POC -> 실제 과제 -> 설치/납품` 전환을 설명하는 문맥이 강하다.
- 따라서 이 축은 납품 전환형 검사 프로젝트로 볼 수 있다.

## CMS

- 배포 테스트, 시각화 변경, overlay 결과, 공장/연구소 이미지 차이, 측정 로직 보정이 중심이다.
- 핵심 문제는 현장 데이터 적합성과 운영 안정화다.
- 따라서 이 축은 배포 후 운영 보정형 프로젝트로 보는 편이 맞다.

## 위키 해석

- 금호타이어는 고객 계정 하나 아래 `납품 전환형(X-ray)`과 `운영 보정형(CMS)` 두 트랙이 병존하는 구조다.
- 향후에는 고객 계정 관점 페이지에서 공통 인물, 공장/연구소 역할, 일정 축을 합치고, 세부 기술/산출물은 하위 프로젝트 문서에서 관리하는 방식이 적절하다.
