# Government RnD Section Skeletons

## 배경 및 필요성

```markdown
{산업/시장}은(는) {변화/성장/압력}이 커지고 있으나, 현재 {현장/공정/설비} 운영은 {구조적 한계}로 인해 {운영상 문제}가 반복되고 있음.

특히 {핵심 문제}는 {구체적 영향}으로 이어지며, 이는 단순 운영 편의의 문제가 아니라 {사업/기술/경쟁력 관점의 의미}를 가짐.

따라서 본 과제는 {해결 방향}을 통해 {최종적으로 달성할 운영 상태}를 구현하는 것을 목표로 함.
```

## 최종 목표

```markdown
본 과제의 최종 목표는 {대상 공정/설비/업무}의 {핵심 문제}를 해결하기 위해, {데이터/문서/운영 지식}를 통합한 {플랫폼/AI/에이전트/시스템}를 개발하고 실증하는 것임.
```

## 세부 목표

```markdown
1. {데이터 통합 목표}
2. {이상 탐지/상황 인지 목표}
3. {원인 분석/예측 목표}
4. {조치 추천/제어 목표}
5. {실증 및 고도화 목표}
```

## 참여기관 역할

```markdown
{기관명}은(는) {보유 기술/전문성}을 기반으로 {담당 모듈} 개발을 담당하며, {검증/연계/실증 기여}를 통해 전체 과제 성능 고도화에 기여함.
```

## 수요기업 역할

```markdown
수요기업 {기관명}은(는) {대상 현장/공정/라인}의 실제 운영 문제를 제시하고, {데이터 종류} 및 {실증 환경}을 제공하며, 개발 결과의 현장 적용 가능성과 도입 효과를 검증함.
```

## 사업화/확산

```markdown
본 과제 결과물은 {1차 확산 대상}을 시작으로 {2차 확산 대상}, {3차 확산 대상}으로 확산 가능한 {패키지/서비스/운영 모델}로 고도화할 계획임.

특히 {고객군/계열사/유사 공정}에 대해 {재사용 가능한 배포 단위}로 전환함으로써, 단일 실증에 그치지 않는 확산 구조를 확보하고자 함.
```

## 보안/운영

```markdown
본 과제는 {폐쇄망/On-Premise/민감데이터} 환경을 전제로 하므로, 역할 기반 접근통제, 로그 기록, 승인 기반 실행, 롤백 절차를 포함한 운영 구조를 적용함.
```
