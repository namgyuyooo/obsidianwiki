# RTM_YNG Wiki Build Plan

## 목표

- `RTM_YNG` Drive 탐색 결과를 단순 증적 저장이 아니라 실제 위키 자산으로 전환한다.
- 대표본이 강한 프로젝트부터 개별 프로젝트 문서 체계를 만들고, 나머지는 증적 기반 후보 상태로 관리한다.

## 현재 완료 상태

- Drive 재귀 탐색 및 대표본 지도 작성 완료
- `Sources`, `Evidence Log`, `Change Log`, `Conflict Register` 생성 완료
- 공용 위키 본문 `[[RTM_YNG_Project_Knowledge_Base_2026-04-21]]` 생성 완료
- 강한 대표본 6개 개별 페이지 생성 완료
- 2차 확장 페이지 생성 완료:
  - [[RTM_YNG_ITM_반도체]]
  - [[RTM_YNG_아사히카세이]]
  - [[RTM_YNG_DMT_POC]]
  - [[RTM_YNG_ZEUS_POC]]
  - [[RTM_YNG_반도체_공정제어_멀티에이전트]]
  - [[RTM_YNG_화승_AI바우처]]
  - [[RTM_YNG_제우스_픽셀_AI바우처]]
  - [[RTM_YNG_한양대_반도체_자율지능형_에이전트]]

## 남은 작업

### 1. 개별 프로젝트 위키 심화

- 목적:
  - 이미 생성된 개별 프로젝트 문서를 실제 운영 위키 수준으로 고도화
- 우선 대상:
  - 산업현장 에이전트
  - 서울반도체
  - 탈레스
  - 케타포
  - 경동나비엔
  - HPP
- 완료 기준:
  - 각 프로젝트마다 `Project Overview`, `KPI`, `Budget/Revenue`, `Risks`, `Version Notes`, `Sources` 연결이 유기적으로 유지됨

### 2. 구조화 표 정리

- 목적:
  - 문장형 지식을 검색 가능한 표로 재정리
- 필요 표:
  - 프로젝트별 KPI 표
  - 프로젝트별 예산/재원 표
  - 프로젝트별 일정/마일스톤 표
  - 프로젝트별 리스크 표
- 완료 기준:
  - 대표본 기준 수치와 버전 불확실 수치가 분리 표기됨

### 3. 비교본 반영 확대

- 목적:
  - 대표본만이 아니라 비교본 차이를 위키에도 반영
- 우선 대상:
  - 서울반도체
  - 산업현장 에이전트
  - 경동나비엔 후속 `poc 제안서`
  - HPP 중간본 추가
- 완료 기준:
  - `Change Log`의 핵심 차이가 개별 프로젝트 페이지 `Notes` 또는 `Version Notes`에 반영됨

### 4. HWP/HWPX 확보 및 파싱

- 목적:
  - 이미 확보한 `hwp/hwpx` 원문을 포함해 대표본/비교본 대조를 정밀화
- 우선 대상:
  - 산업현장 에이전트
  - 서울반도체
  - 제우스 / 픽셀
  - 화승 AI바우처
  - 한양대 / 반도체 자율지능형 에이전트
  - 경동나비엔 작성중 문서
- 완료 기준:
  - `rhwp` 또는 다른 확보 경로로 본문 텍스트 추출 성공
  - `Change Log`와 `Conflict Register`에 실제 diff 반영

### 5. 이미지/발표자료 기반 보강

- 목적:
  - `pptx`, `png`, `jpg`, `zip`, `mp4` 기반 증거를 위키 지식으로 연결
- 우선 대상:
  - 경동나비엔
  - HPP
  - 아사히카세이
  - DMT_POC
  - ITM 반도체
- 완료 기준:
  - 이미지/발표자료가 단순 첨부가 아니라 공정, 검사 포인트, 성능 조건 설명으로 연결됨

### 6. 2차 프로젝트군 확장

- 목적:
  - 현재 생성된 2차 프로젝트군을 추가 심화하거나 새 후보를 편입
- 대상:
  - 아사히카세이
  - ITM 반도체
  - DMT_POC
  - ZEUS
  - 반도체 공정제어 멀티에이전트
  - 화승 AI바우처
  - 한양대 / 반도체 자율지능형 에이전트
- 완료 기준:
  - 공용 후보 상태가 아닌 개별 페이지 + 버전 메모 또는 증적 링크 확보

## 실행 순서

1. 개별 프로젝트 페이지와 공용 문서 상태를 동기화
2. 각 프로젝트의 KPI/예산/리스크 표를 고도화
3. 비교본 차이를 `Version Notes`와 `Change Log`에 반영
4. 확보한 `hwp/hwpx` 원문을 정밀 비교
5. 이미지/발표자료 기반 지식을 추가
6. 남은 중간 강도 프로젝트군을 계속 편입

## 이번 세션 우선순위

### P1

- `화승` `0318~최종본` 사업비/기관현황 diff 정밀화
- `한양대` 조상 `hwp/hwpx`와 `0310 pdf` 연결 심화

### P2

- `경동나비엔` 보조 `txt/png`와 작성중 `hwp/hwpx` 연결
- `ZEUS/픽셀` `최종 hwpx vs 중간보고` diff

### P3

- `산업현장 에이전트`, `서울반도체` 세부 표 diff
- `HPP`, `아사히카세이`, `DMT_POC` 시각 증거 보강

## 작업 규칙

- 대표본 기준 정보와 버전 비교 정보는 분리해서 쓴다.
- 불확실한 수치는 `Conflict Register`와 링크한다.
- 공용 문서는 인덱스 역할, 실제 지식은 프로젝트 페이지로 내린다.
