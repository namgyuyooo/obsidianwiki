# RTM_YNG Wiki Build Plan

## 목표

- `RTM_YNG` Drive 탐색 결과를 단순 증적 저장이 아니라 실제 위키 자산으로 전환한다.
- 대표본이 강한 프로젝트부터 개별 프로젝트 문서 체계를 만들고, 나머지는 증적 기반 후보 상태로 관리한다.

## 현재 완료 상태

- Drive 재귀 탐색 및 대표본 지도 작성 완료
- `Sources`, `Evidence Log`, `Change Log`, `Conflict Register` 생성 완료
- 공용 위키 본문 `[[RTM_YNG_Project_Knowledge_Base_2026-04-21]]` 생성 완료
- 강한 대표본 6개 개별 페이지 생성 완료
- 2차 확장 페이지 생성 완료:
  - [[RTM_YNG_ITM_반도체]]
  - [[RTM_YNG_아사히카세이]]
  - [[RTM_YNG_DMT_POC]]
  - [[RTM_YNG_ZEUS_POC]]
  - [[RTM_YNG_반도체_공정제어_멀티에이전트]]
  - [[RTM_YNG_화승_AI바우처]]

## 남은 작업

### 1. 개별 프로젝트 위키 분리

- 목적:
  - 공용 Knowledge Base의 내용을 프로젝트별 독립 문서로 분리
- 우선 대상:
  - 산업현장 에이전트
  - 서울반도체
  - 탈레스
  - 케타포
  - 경동나비엔
  - HPP
- 완료 기준:
  - 각 프로젝트마다 최소 `Project Overview`, `KPI`, `Budget/Revenue`, `Risks`, `Sources` 연결이 보이는 단일 페이지 확보

### 2. 구조화 표 정리

- 목적:
  - 문장형 지식을 검색 가능한 표로 재정리
- 필요 표:
  - 프로젝트별 KPI 표
  - 프로젝트별 예산/재원 표
  - 프로젝트별 일정/마일스톤 표
  - 프로젝트별 리스크 표
- 완료 기준:
  - 대표본 기준 수치와 버전 불확실 수치가 분리 표기됨

### 3. 비교본 반영 확대

- 목적:
  - 대표본만이 아니라 비교본 차이를 위키에도 반영
- 우선 대상:
  - 서울반도체
  - 산업현장 에이전트
  - 경동나비엔 후속 `poc 제안서`
  - HPP 중간본 추가
- 완료 기준:
  - `Change Log`의 핵심 차이가 개별 프로젝트 페이지 `Notes` 또는 `Version Notes`에 반영됨

### 4. HWP/HWPX 확보 및 파싱

- 목적:
  - 현재 막혀 있는 `hwp/hwpx` 원문을 확보해 대표본/비교본 대조를 정밀화
- 우선 대상:
  - 산업현장 에이전트
  - 서울반도체
  - 대학연스타트업 과제
  - 화승 AI바우처
- 완료 기준:
  - `rhwp` 또는 다른 확보 경로로 본문 텍스트 추출 성공
  - `Change Log`와 `Conflict Register`에 실제 diff 반영

### 5. 이미지/발표자료 기반 보강

- 목적:
  - `pptx`, `png`, `jpg`, `zip`, `mp4` 기반 증거를 위키 지식으로 연결
- 우선 대상:
  - 경동나비엔
  - HPP
  - 아사히카세이
  - DMT_POC
  - ITM 반도체
- 완료 기준:
  - 이미지/발표자료가 단순 첨부가 아니라 공정, 검사 포인트, 성능 조건 설명으로 연결됨

### 6. 2차 프로젝트군 확장

- 목적:
  - 현재 강한 6개 프로젝트 외 중간 강도 후보를 위키 편입
- 대상:
  - 대학연스타트업 과제
  - 아사히카세이
  - ITM 반도체
  - DMT_POC
  - ZEUS
  - 반도체 공정제어 멀티에이전트
  - 화승 AI바우처
- 완료 기준:
  - 공용 후보 상태가 아닌 최소 개별 요약 페이지 생성

## 실행 순서

1. 강한 대표본 6개를 개별 프로젝트 위키로 분리
2. 각 프로젝트의 KPI/예산/리스크 표를 추가
3. 비교본 차이를 `Version Notes`로 반영
4. `hwp/hwpx` 확보 경로를 열어 정밀 비교
5. 이미지/발표자료 기반 지식을 추가
6. 중간 강도 프로젝트군으로 확장

## 이번 세션 우선순위

### P1

- 산업현장 에이전트 개별 페이지 생성
- 서울반도체 개별 페이지 생성
- 탈레스/케타포 개별 페이지 생성

### P2

- 경동나비엔/HPP 개별 페이지 생성
- KPI/예산 표 추가

### P3

- `hwp/hwpx` 확보 경로 재시도
- 2차 프로젝트군 확장

### P4

- `한양대`, `화승`, `2024 AI바우처 / 제우스 / 픽셀`의 `hwpx` 우회 확보
- 신규 2차 확장 프로젝트의 `Sources/Evidence` 정리 고도화

### P5

- `화승` 영상/이미지 결과 세트와 사업계획서 연결
- `한양대` 응답 안정화 후 본문 추출 재시도

## 작업 규칙

- 대표본 기준 정보와 버전 비교 정보는 분리해서 쓴다.
- 불확실한 수치는 `Conflict Register`와 링크한다.
- 공용 문서는 인덱스 역할, 실제 지식은 프로젝트 페이지로 내린다.
