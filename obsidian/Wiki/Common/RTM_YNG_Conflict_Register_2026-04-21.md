### C-007 제우스 매출기반 파일명 분기와 성능 기준선

- 프로젝트: 2024 AI바우처 / 제우스 / 픽셀
- 현재 기준:
  - `제우스 중간보고 hwpx`는 목표형 표현(`객체 인식 0.95 이상`, `추정정확도 0.9 이상`) 중심
  - `제우스 최종 hwpx`는 실적형 표현(`평균 오류율 0.0204`, `추정정확도 변환 성능 100%`, `주관기관 매출 15억원 확보`) 중심
  - 추가 직접 추출 기준으로는 제우스 최종 `hwpx`에서 Hubble 객체 탐지/추적 `99.42%`, Apollo 공정 상태 예측 `100%`, 리드타임 `112시간 -> 35.2시간`, 약 `68.57%` 단축이 확인됨
  - 픽셀 최종 `hwpx`는 제우스의 단순 부속이 아니라 `미검율 0.15%`, `과검율 0.91%`, `Takt time 1.354ms`, Total Accuracy `99.962%`를 가진 독립 최종보고서 성격이 강함
- 충돌 가능성:
  - 제출 메모에는 `매출기반O/X` 분기가 적혀 있지만, 실제 확보한 Drive 실물 파일명은 `2.최종보고서_(주)제우스_매출기반 .hwpx`로 확인되어 파일명 정합성 이슈가 있음
  - `TTA_제우스 설명 자료`의 시험목표와 `중간보고`의 목표표, `최종보고`의 최종 성능표가 같은 지표 체계인지 아직 확정되지 않음
  - 픽셀 발표자료의 수주/시장효과 전망과 픽셀 최종보고서의 달성 성과를 같은 성격의 수치로 섞으면 과대 해석이 생김
- 상태: 보류
- 다음 확인:
  - `매출기반X` 실물 파일 확보
  - `중간보고 / TTA / 최종보고`의 객체인식·추정정확도 표를 같은 산식 기준으로 매핑
  - 픽셀/제우스가 동일 바우처 하위 트랙인지, 같은 제출 주간의 별도 과제 묶음인지 Drive 폴더 계보와 최종 제출 메모로 재확인

### C-005 경동나비엔 POC 성능 비교 조건 차이

- 프로젝트: 경동나비엔 / RTM
- 현재 기준:
  - 1차 POC: 11개 검사포인트, 1,642장, 정확도 100%
  - 2차 POC: 176개 포인트, 탐지 성능 97.2%
- 충돌 가능성:
  - 같은 성능 비교처럼 보이지만 데이터셋과 촬영 조건이 달라 단순 비교가 어렵다
  - 선행 `Pre-PoC`는 이미지 검사보다 체결 로봇 동작시간 기반 이상탐지 가능성 검토 문서라, 현장 비전 PoC와 직접 같은 축의 성능 비교로 읽으면 안 된다
  - 후속 메일 발췌에서는 `균열 사전 예측`, `금형 마모 상태 진단`, `칩 찍힘 예방`, `평탄 이상 조기 확인`처럼 더 넓은 품질 예지보전 요구가 등장해, 단순 누락 검사 KPI와 과제형 KPI가 달라질 가능성이 있다
- 상태: 보류
- 다음 확인:
  - `Pre-PoC`, 1차 POC, 2차 POC의 평가모수와 ROI 조건을 분리 기록
  - 작성중 `1-1/1-2 hwp/hwpx`에서 실제 과제 KPI가 어느 범위까지 확장됐는지 확인

### C-006 HPP 성능 수치의 문서 간 일치 여부

- 프로젝트: HPP
- 현재 기준: PDF에서 Accuracy 0.9487, Precision 0.9, Recall 1, F1-score 0.9474
- 충돌 가능성:
  - 대응 `pptx`에서는 현재까지 동일 수치가 유지되며 직접 충돌은 확인되지 않았다
  - 다만 `pptx`에만 데이터 규모(`정상 90개`, `불량 시료 15종`, 출력물 `정상 21장 / 불량 18장`)와 추진 마일스톤이 더 많이 남아 있어, 향후 중간본과 비교하면 평가셋 정의 차이가 드러날 수 있다
  - 초기 `HPP_final`은 Doctor GAP 중심 `450`개 라벨링 데이터 기반 기술검토 문서라, 최종 4대 과제 통합 보고서와 비교할 때 범위 차이를 성능 충돌로 오해할 수 있다
- 상태: 보류
- 다음 확인:
  - `HPP_final` 중간본과 비교해 평가셋, 마일스톤, 예산 산정 근거의 일치 여부 확인
