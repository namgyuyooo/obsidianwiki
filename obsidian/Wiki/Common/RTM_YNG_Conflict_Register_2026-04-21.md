### C-003 탈레스 예산 구조와 인건비 처리

- 프로젝트: Trust my Tech / 탈레스
- 현재 기준: `vf` 본문에서 정부지원 100백만원, 총사업비 143백만원 구조가 보임
- 충돌 가능성:
  - `인건비 제외본`은 정부지원 `192,480,000원`, 총사업비 `274,980,000원`, 현금 `27,510,000원`, 현물 `54,990,000원`으로 읽혀 `vf`와 예산 기준이 실질적으로 다르다
  - 과제번호도 `vf`의 `20446514`와 `인건비 제외본`의 `20286906`으로 달라 동일 문서의 단순 비용 조정본이 아닐 수 있다
  - 제품/범위 서술도 `TS Agent` 중심과 `Hubble` 보안형 제조공정 최적화 중심으로 갈라지는 흔적이 있다
- 상태: 확정
- 다음 확인:
  - 같은 공모의 변형본인지, 별도 제안 트랙인지 폴더 계보와 메타데이터 재확인

### C-004 케타포 목표 수치와 사업 범위 버전차

- 프로젝트: AI바우처 / 케타포

  - `0.5`와 `최종취합`은 같은 프로젝트 축으로 보이나, 기대성과 표와 KPI 요약부가 정리되면서 숫자 표현이 다듬어졌을 가능성이 있다
- 상태: 보류
- 다음 확인:

### C-005 경동나비엔 POC 성능 비교 조건 차이

- 프로젝트: 경동나비엔 / RTM
- 현재 기준:
  - 1차 POC: 11개 검사포인트, 1,642장, 정확도 100%
  - 2차 POC: 176개 포인트, 탐지 성능 97.2%
- 충돌 가능성:
  - 같은 성능 비교처럼 보이지만 데이터셋과 촬영 조건이 달라 단순 비교가 어렵다
  - 선행 `Pre-PoC`는 이미지 검사보다 체결 로봇 동작시간 기반 이상탐지 가능성 검토 문서라, 현장 비전 PoC와 직접 같은 축의 성능 비교로 읽으면 안 된다
- 상태: 보류
- 다음 확인:
  - `Pre-PoC`, 1차 POC, 2차 POC의 평가모수와 ROI 조건을 분리 기록

### C-006 HPP 성능 수치의 문서 간 일치 여부

- 프로젝트: HPP
- 현재 기준: PDF에서 Accuracy 0.9487, Precision 0.9, Recall 1, F1-score 0.9474
- 충돌 가능성:
  - 대응 `pptx`에서는 현재까지 동일 수치가 유지되며 직접 충돌은 확인되지 않았다
  - 다만 `pptx`에만 데이터 규모(`정상 90개`, `불량 시료 15종`, 출력물 `정상 21장 / 불량 18장`)와 추진 마일스톤이 더 많이 남아 있어, 향후 중간본과 비교하면 평가셋 정의 차이가 드러날 수 있다
  - 초기 `HPP_final`은 Doctor GAP 중심 `450`개 라벨링 데이터 기반 기술검토 문서라, 최종 4대 과제 통합 보고서와 비교할 때 범위 차이를 성능 충돌로 오해할 수 있다
- 상태: 보류
- 다음 확인:
  - `HPP_final` 중간본과 비교해 평가셋, 마일스톤, 예산 산정 근거의 일치 여부 확인
