# Government RnD Wiki Operating Guide

## 목표

개별 정부 R&D 문서를 읽고 끝내지 않고, 향후 다른 과제에도 재사용 가능한 공통 자료와 문장 자산으로 축적하는 운영 절차입니다.

## 기본 흐름

1. 기존 위키에서 관련 프로젝트와 공통 자산을 먼저 확인합니다.
2. Google Drive에서 공고문, 양식, 대표본, 비교본을 찾습니다.
3. `hwp`와 `hwpx`를 포함해 실제 항목 표현을 확인합니다.
4. 프로젝트 전용 내용과 공통 자산 후보를 분리합니다.
5. 공통 자산은 이 `Government_RnD_*` 문서군에 승격합니다.
6. 프로젝트별 문서는 `Sources`, `Evidence Log`, `Conflict Register`, `Change Log`를 유지합니다.

## 프로젝트 전용 vs 공통 자산 구분

### 프로젝트 전용

- 특정 고객사 고유 공정
- 특정 예산 수치
- 특정 장비 스펙
- 특정 실증 일정

### 공통 자산

- 항목별 문장 구조
- 역할 서술 골격
- 정량 KPI 표현 방식
- 보안/운영/확산 서술 패턴
- 대표본 선정 기준

## 위키 반영 규칙

- 원문 근거는 프로젝트 `Evidence Log`에 남깁니다.
- 공통 문장은 여기 `Common` 문서로 요약 승격합니다.
- 충돌 값은 절대 공통 자산에 바로 넣지 말고 프로젝트 `Conflict Register`에 먼저 등록합니다.
- 공통 자산에는 `충돌 없는 패턴` 또는 `충돌을 감안해도 반복 사용 가능한 서술 구조`만 남깁니다.

## 추천 산출물

- 프로젝트별:
  - `Sources.md`
  - `Evidence Log.md`
  - `Conflict Register.md`
  - `Change Log.md`
- 공통:
  - `Government_RnD_HWP_Expression_Bank`
  - `Government_RnD_Participant_Role_Guide`
  - `Government_RnD_Writing_Style_Guide`

## 다음 과제에서 바로 쓰는 방법

- 새 과제 시작 시 `Government_RnD_Reusable_Wiki_Hub`를 먼저 읽습니다.
- 역할 문장은 `Participant_Role_Guide`에서 골격을 가져옵니다.
- 문체와 항목명은 `Writing_Style_Guide`, `HWP_Expression_Bank` 기준으로 맞춥니다.
- 실제 문장 채움은 새 과제의 Evidence Log에서만 가져옵니다.
