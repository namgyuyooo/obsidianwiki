# Conflict Register

## Open Conflicts
- 현재 Common 레벨에서 별도 유지 중인 명시적 공통 conflict는 없음

## Routing Notes
- 프로젝트 여부가 불명확한 혼합 채널 대화는 [[Wiki/Common/Slack_Project_Intake/Conflict_Register]]에서 보류 후 재분류
- Common 문서 정비, 허브 보강, 구조 점검은 [[Wiki/Common/Action_Items]]와 [[Wiki/Common/Decisions]]에서 먼저 처리
- 승인 로그, 위키 승격 로그, 병합 메타데이터는 Conflict_Register에 남기지 않음

## Capture Rule
- 여러 프로젝트나 공통 운영 자산에 동시에 영향을 주는 명시적 상충 기준만 여기서 관리합니다.
- 단순 운영 TODO나 구조 메모는 Common conflict가 아니라 운영 액션으로 처리합니다.
