# RTM_YNG Project Knowledge Base

## 목적

- 이 문서는 `RTM_YNG` Drive에서 확보한 대표본과 비교본을 바탕으로, 실제 프로젝트 지식을 위키 형태로 바로 읽을 수 있게 정리한 본문이다.
- `Sources`, `Evidence Log`, `Change Log`, `Conflict Register`가 증적과 이력을 관리한다면, 이 문서는 프로젝트 개요와 핵심 정보를 빠르게 파악하기 위한 운영 위키다.

## 1. 산업현장 에이전트_연구개발

- 개별 페이지: [[RTM_YNG_산업현장_에이전트_연구개발]]

### Project Overview

- 과제명은 제조현장 예지보전과 지능형 제조를 위한 데이터-지식 융합 기반 자율운영 AI 에이전트 개발로 정리된다.
- 과제번호는 `RS-2026-25553046`이며, 수행기간은 `2026.04.01 ~ 2028.12.31`이다.
- 핵심 방향은 공정 지식과 실시간 설비 데이터를 통합해 이상/고장을 식별하고, 현장 작업자와 협업해 진단과 조치를 수행하는 자율운영형 AI 에이전트를 만드는 것이다.

### KPI

- 설비 가동률 `99% 이상`
- 유효 학습데이터 `50만건 이상`
- 상황인지/이상탐지 정확도 `93% 이상`
- 성능 개선 `40%`
- 잔존수명 예측 신뢰도 `95% 이상`
- 정비 권고 적중률 `80% 이상`
- 추천/예측 오차 `5% 이하`

### Budget

- 총 연구개발비 `6,589,554천원`
- 정부지원연구개발비 `5,000,000천원`
- 기관부담연구개발비 `1,589,554천원`

### Risks

- `0414~0416_RTM 취합`과 최종본 사이 KPI와 예산 표현이 달라졌을 가능성이 있어 `hwp` 파싱 전에는 기준 수치를 대표본 기준으로만 써야 한다.
- 데이터-지식 융합과 자율운영 에이전트라는 개념 범위가 넓어, 실제 PoC 범위와 최종 사업 범위를 구분해 기록할 필요가 있다.

## 2. AI 응용제품 신속 상용화 / 서울반도체

- 개별 페이지: [[RTM_YNG_서울반도체_AI_응용제품_신속_상용화]]

### Project Overview

- 문서는 `AI 응용제품 신속 상용화 지원사업` 사업계획서다.
- LED/반도체 제조 공정 맥락에서 공정/설비 데이터가 분산되어 있어 AI 기반 통합 운영 체계가 필요하다는 문제 정의가 강하다.
- 제품/서비스 개요, 현행 공정, 도입 계획, 상용화 전략, 인력 운영, 보안/안전, 예산 세부내역까지 포함한 완결형 사업계획서에 가깝다.

### Budget

- 정부지원 `100백만원`
- 총사업비 `143백만원`
- 민간부담 현금 `15백만원`
- 민간부담 현물 `25백만원`

### Knowledge Points

- 단순 모델 개발보다 제조공정 최적화와 AI 도입 체계 구축이 중심이다.
- 사업비, 인력, 안전, 보안 항목까지 함께 들어 있어 실제 제출본 기준 지식원으로 쓰기 적합하다.

### Risks

- `사업계획서_서울반도체.hwpx`, `TalkFile_서울반도체_사업계획서(사업비).hwp`, `사업계획_최종합의서.pdf` 사이 사업비 표현 차이가 있을 수 있다.
- KPI 표는 후속 `hwp/hwpx` 확보 후 별도 표준화가 필요하다.

## 3. Trust my Tech / 탈레스

- 개별 페이지: [[RTM_YNG_탈레스_Trust_my_Tech]]

### Project Overview

- 신청 주체는 `고려대 세종산학협력단`, 과제번호는 `20446514`로 읽힌다.
- 제품/산출물 축은 `TS Agent`로 제시되며, 반도체 장비용 데이터-지식 융합 기반 자율운영 AI 에이전트 성격이 강하다.
- 보안 민감 제조현장, 온프레미스 배포, 역할 기반 접근통제, 접근 로그, 문서 보호, 감사 추적, 글로벌 제안 확대가 핵심 키워드다.

### Budget

- `vf` 대표본 기준 정부지원 `100백만원`
- `vf` 대표본 기준 총사업비 `143백만원`

### Knowledge Points

- 이 프로젝트는 일반 AI 자동화보다 보안형 제조 현장 적용성이 더 중요하게 다뤄진다.
- 해외 고객 제안과 글로벌 협업 확장 맥락이 같이 제시되어 있어, 기술 제안과 사업 확장 논리가 동시에 존재한다.

### Risks

- `인건비 제외본`은 과제번호 `20286906`, 정부지원 `192,480,000원`, 총사업비 `274,980,000원`으로 읽혀 `vf`와 실제 충돌한다.
- `TS Agent` 중심 서술과 `Hubble` 보안형 제조공정 최적화 서술이 갈라져 있어, 같은 제안의 변형인지 다른 트랙인지 추가 판별이 필요하다.

## 4. AI바우처 / 케타포

- 개별 페이지: [[RTM_YNG_케타포_AI바우처]]

### Project Overview

- 주관기관은 `㈜케이타운포유`, 참여기관은 `㈜알티엠`이다.
- 수행기간은 `2026년 5월 ~ 2026년 11월 30일`이다.
- 과제는 글로벌 K-POP 팬덤 커머스의 발주, CS, 물류 운영 최적화를 위한 멀티에이전트 개발로 정리된다.

### Solution Structure

- 발주 예측 에이전트
- CS 자동화 액션 에이전트
- 패킹 알고리즘 에이전트
- 커머스 모니터링 에이전트
- 중앙 오케스트레이터

### KPI

- 수요예측 오차 `P80 APE <= 20%`

### Revenue / Value

- 매출, 수출, 응답 대기시간 단축, CS 반복문의 자동화 비율이 기대성과 축으로 제시된다.
- PoC 기반으로 `CS FAQ 봇`, `발주 예측 모델`, `빈패킹 회귀 모델`을 거쳐 장기적으로 자율 CS/자율 발주/강화학습 기반 빈패킹으로 확장하는 로드맵이 있다.

### Risks

- `0.3`은 템플릿 흔적이 많아 메타정보 기준본으로 쓰기 어렵다.
- `0.5`와 `최종취합`은 방향 자체는 같지만 기대성과와 KPI 요약 표현이 다듬어졌을 가능성이 있어 숫자 표는 버전 대조가 더 필요하다.

## 5. 경동나비엔 / RTM

- 개별 페이지: [[RTM_YNG_경동나비엔_RTM]]

### Project Overview

- 대표본은 경동나비엔 대상 AI 솔루션 제안서이며, 볼트 및 클립 누락 검사 과제를 핵심 범위로 다룬다.
- RTM 회사 소개와 Manufacturing AI 역량, 기존 고객 사례가 함께 들어 있는 영업형 제안서 구조다.
- 선행 `Pre-PoC` 단계에서는 체결 로봇 동작시간 데이터 기반 이상탐지 가능성을 먼저 검토했고, 이후 비전 기반 검사와 구축안으로 확장됐다.

### Performance

- 1차 POC: `11개 검사포인트`, `1,642장 이미지`, `정확도 100%`
- 2차 POC: `176개 포인트`, `탐지 성능 97.2%`

### Technical Scope

- 기존 AWS Lookout for Vision 기반 운영을 RTM 학습/추론 솔루션으로 대체 또는 전환하는 흐름
- ROI 자동 지정
- 제품별 검사영역 설정
- PLC 연동
- 사내 추론서버 활용
- 신규 고정식 광학 시스템 도입

### Budget

- 1라인 총 견적 `167,100,000원`
- 솔루션 라이선스 `25,000,000원`
- 개발 `112,100,000원`
- HW `30,000,000원`
- 라인 확장안 `150,000,000원`

### Risks

- `Pre-PoC`, 1차 POC, 2차 POC는 평가 대상과 데이터셋이 달라 성능을 같은 축으로 단순 비교하면 안 된다.
- `png`, `txt`, 후속 `poc 제안서_00.pptx`를 더 열어 검사 포인트와 ROI 규칙을 세분화할 필요가 있다.

## 6. HPP

- 개별 페이지: [[RTM_YNG_HPP]]

### Project Overview

- HPPK 4대 AI 도입을 통한 생산성 개선 프로젝트 추진 계획 및 기술 판단 보고서 성격을 갖는다.
- 핵심 과제는 `E-ring 누락`, `Doctor GAP 측정`, `Print 출력 검사`, `Printer 외관 검사`의 4대 라인업이다.
- 초기 `HPP_final`은 Doctor GAP 중심의 기술검토 의견보고서였고, 최종본으로 가면서 4대 과제를 포괄하는 프로그램형 제안으로 확장됐다.

### Performance

- Print 검사 기준 `Accuracy 0.9487`
- `Precision 0.9`
- `Recall 1`
- `F1-score 0.9474`

### Operating Detail

- 대응 `pptx`에는 `2025.11.21` 초기 미팅, `2025.12.04` 현장 촬영/광학 검토, `2025.12.31` AI 파이프라인 개발, `2026.01.28` 2단계 완료가 더 선명하게 기록돼 있다.
- `pptx`에는 데이터 규모도 남아 있다.
  - 정상 제품 `90개`
  - 불량 시료 `15종`
  - 출력물 검사 `정상 21장 / 불량 18장`

### Roadmap / Value

- Quick Win
- Process Governance
- Quality Assurance
- ROI
- 2단계 확장과 AI 기반 제조 DX 로드맵

### Budget

- 후속 제안 예산 `약 2.5억 원`

### Risks

- PDF와 최종 `pptx`는 큰 수치가 일치하지만, 초기 `HPP_final`은 Doctor GAP 중심 `450`개 라벨링 데이터 기반 검토본이라 범위가 다르다.
- 평가셋과 데이터 수량은 버전별로 구분해 관리해야 한다.

## 7. ZEUS_POC / BD 이물탐지

- 개별 페이지: [[RTM_YNG_ZEUS_POC]]

### Project Overview

- `ZEUS_POC 결과보고서`는 `CT(Coater)`와 `BD(Bonder)` NVR 영상 기반 편심·이물 감지 가능성을 검토한 기술 타당성 보고서다.
- 결론은 현재 촬영 조건에서는 이물 감지도, 편심 감지도 직접 도입 가능한 수준이 아니라는 쪽에 가깝다.
- 즉, 이 프로젝트는 성공 사례보다는 실패 조건과 다음 하드웨어 요구사항을 정리한 문서로 읽는 편이 맞다.

### Key Findings

- 이물은 전 구간에서 검출되지 않아 추적 자체가 불가능하다고 정리된다.
- 편심은 사선 촬영, 해상도, 고속 구간 흔적 소실 때문에 안정적 감지가 어렵다고 제시된다.
- 대안으로 수직 촬영, 해상도 상향, 레이저 프로브 등 하드웨어 보강이 제안된다.

### Risks

- 보고된 총 영상 수 `49개`와 세부 합계 `CT 20 + BD 19 = 39`가 맞지 않아 원문 재확인이 필요하다.
- `2024` 바우처 계열 `hwpx`를 아직 열지 못해 ZEUS 계열 과제가 어디까지 이어지는지는 추가 확인이 필요하다.

## 8. 반도체 공정제어 멀티에이전트

- 개별 페이지: [[RTM_YNG_반도체_공정제어_멀티에이전트]]

### Project Overview

- 과제명은 `소부장 온톨로지 연계 온디바이스 기반, 수율 극대화용 선제적 반도체 공정 제어 멀티에이전트 기술 개발`이다.
- 과제번호는 `RS-2026-25523818`, 연구개발기간은 `2026.04.01 ~ 2028.12.31`로 확인된다.
- 세정 공정을 시작점으로 온톨로지, 온디바이스 AI, 멀티에이전트, 피지컬 AI를 결합해 반도체 제조 자율운영 체계를 구축하려는 방향이 명확하다.

### Knowledge Points

- 1차 실증 범위는 세정 공정이며, 이후 식각·증착 등으로 확산하는 전략을 취한다.
- 단일 정확도보다 운영 안정성, 드리프트 대응, OTA, 감사 추적까지 함께 보는 도입형 지표 체계가 강조된다.
- `PSK`, `ZEUS`, 소재·부품사, `S2W`, `성균관대`가 연결되는 밸류체인형 실증 구조로 읽힌다.

### Risks

- 발표자료와 질의응답서 중심으로 읽은 상태라 전체 사업비와 세부 KPI는 아직 더 채워야 한다.
- `한양대` 후보군 `hwpx`와 이 과제가 직접 같은 트랙인지는 추가 확인이 필요하다.

## 9. 화승 AI바우처

- 개별 페이지: [[RTM_YNG_화승_AI바우처]]

### Project Overview

- 과제명은 `믹싱 데이터 기반 자동차용 복합형상 웨더스트립 연속 제조공정 최적화 및 물성 예측 솔루션`이다.
- 주관 수요기업은 `㈜화승코퍼레이션`, 참여 공급기업은 `㈜알티엠`이며, 수행기간은 `2025.05.01 ~ 2025.11.30`이다.
- `EHM` 기반 설비 이상 탐지와 가상계측 기반 물성 예측을 동시에 추진하는 AI바우처 제조 데이터 과제로 읽힌다.

### Budget / KPI

- 총과제비 `430,000천원`
- 정부지원금 `230,000천원`
- 민간부담금 `200,000천원`
- 물성치 추정 정확도 `0.9 이상`
- 설비 데이터 이상탐지 성능 `F1 score 0.9 이상`

### Risks

- 현재 읽힌 문서는 `인건비 삭제` 버전이라 삭제 전 원본과의 차이는 아직 비교되지 않았다.
