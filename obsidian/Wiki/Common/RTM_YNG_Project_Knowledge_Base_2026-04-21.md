# RTM_YNG Project Knowledge Base

## 목적

- 이 문서는 `RTM_YNG` Drive에서 확보한 대표본과 비교본을 바탕으로, 실제 프로젝트 지식을 위키 형태로 바로 읽을 수 있게 정리한 본문이다.
- `Sources`, `Evidence Log`, `Change Log`, `Conflict Register`가 증적과 이력을 관리한다면, 이 문서는 프로젝트 개요와 핵심 정보를 빠르게 파악하기 위한 운영 위키다.

## 1. 산업현장 에이전트_연구개발

- 개별 페이지: [[RTM_YNG_산업현장_에이전트_연구개발]]

### Project Overview

- 과제명은 제조현장 예지보전과 지능형 제조를 위한 데이터-지식 융합 기반 자율운영 AI 에이전트 개발로 정리된다.
- 과제번호는 `RS-2026-25553046`이며, 수행기간은 `2026.04.01 ~ 2028.12.31`이다.
- 핵심 방향은 공정 지식과 실시간 설비 데이터를 통합해 이상/고장을 식별하고, 현장 작업자와 협업해 진단과 조치를 수행하는 자율운영형 AI 에이전트를 만드는 것이다.

### KPI

- 설비 가동률 `99% 이상`
- 유효 학습데이터 `50만건 이상`
- 상황인지/이상탐지 정확도 `93% 이상`
- 성능 개선 `40%`
- 잔존수명 예측 신뢰도 `95% 이상`
- 정비 권고 적중률 `80% 이상`
- 추천/예측 오차 `5% 이하`

### Budget

- 총 연구개발비 `6,589,554천원`
- 정부지원연구개발비 `5,000,000천원`
- 기관부담연구개발비 `1,589,554천원`

### Risks

- `0414~0416_RTM 취합`과 최종본 사이 KPI와 예산 표현이 달라졌을 가능성이 있어 `hwp` 파싱 전에는 기준 수치를 대표본 기준으로만 써야 한다.
- 데이터-지식 융합과 자율운영 에이전트라는 개념 범위가 넓어, 실제 PoC 범위와 최종 사업 범위를 구분해 기록할 필요가 있다.

## 2. AI 응용제품 신속 상용화 / 서울반도체

- 개별 페이지: [[RTM_YNG_서울반도체_AI_응용제품_신속_상용화]]

### Project Overview

- 문서는 `AI 응용제품 신속 상용화 지원사업` 사업계획서다.
- LED/반도체 제조 공정 맥락에서 공정/설비 데이터가 분산되어 있어 AI 기반 통합 운영 체계가 필요하다는 문제 정의가 강하다.
- 제품/서비스 개요, 현행 공정, 도입 계획, 상용화 전략, 인력 운영, 보안/안전, 예산 세부내역까지 포함한 완결형 사업계획서에 가깝다.

### Budget

- 정부지원 `100백만원`
- 총사업비 `143백만원`
- 민간부담 현금 `15백만원`
- 민간부담 현물 `25백만원`

### Knowledge Points

- 단순 모델 개발보다 제조공정 최적화와 AI 도입 체계 구축이 중심이다.
- 사업비, 인력, 안전, 보안 항목까지 함께 들어 있어 실제 제출본 기준 지식원으로 쓰기 적합하다.

### Risks

- `사업계획서_서울반도체.hwpx`, `TalkFile_서울반도체_사업계획서(사업비).hwp`, `사업계획_최종합의서.pdf` 사이 사업비 표현 차이가 있을 수 있다.
- KPI 표는 후속 `hwp/hwpx` 확보 후 별도 표준화가 필요하다.

## 3. Trust my Tech / 탈레스

- 개별 페이지: [[RTM_YNG_탈레스_Trust_my_Tech]]

### Project Overview

- 신청 주체는 `고려대 세종산학협력단`, 과제번호는 `20446514`로 읽힌다.
- 제품/산출물 축은 `TS Agent`로 제시되며, 반도체 장비용 데이터-지식 융합 기반 자율운영 AI 에이전트 성격이 강하다.
- 보안 민감 제조현장, 온프레미스 배포, 역할 기반 접근통제, 접근 로그, 문서 보호, 감사 추적, 글로벌 제안 확대가 핵심 키워드다.

### Budget

- `vf` 대표본 기준 정부지원 `100백만원`
- `vf` 대표본 기준 총사업비 `143백만원`

### Knowledge Points

- 이 프로젝트는 일반 AI 자동화보다 보안형 제조 현장 적용성이 더 중요하게 다뤄진다.
- 해외 고객 제안과 글로벌 협업 확장 맥락이 같이 제시되어 있어, 기술 제안과 사업 확장 논리가 동시에 존재한다.

### Risks

- `인건비 제외본`은 과제번호 `20286906`, 정부지원 `192,480,000원`, 총사업비 `274,980,000원`으로 읽혀 `vf`와 실제 충돌한다.
- `TS Agent` 중심 서술과 `Hubble` 보안형 제조공정 최적화 서술이 갈라져 있어, 같은 제안의 변형인지 다른 트랙인지 추가 판별이 필요하다.

## 5. 경동나비엔 / RTM

- 개별 페이지: [[RTM_YNG_경동나비엔_RTM]]

### Project Overview

- 대표본은 경동나비엔 대상 AI 솔루션 제안서이며, 볼트 및 클립 누락 검사 과제를 핵심 범위로 다룬다.
- RTM 회사 소개와 Manufacturing AI 역량, 기존 고객 사례가 함께 들어 있는 영업형 제안서 구조다.
- 선행 `Pre-PoC` 단계에서는 체결 로봇 동작시간 데이터 기반 이상탐지 가능성을 먼저 검토했고, 이후 비전 기반 검사와 구축안으로 확장됐다.

### Performance

- 1차 POC: `11개 검사포인트`, `1,642장 이미지`, `정확도 100%`
- 2차 POC: `176개 포인트`, `탐지 성능 97.2%`

### Technical Scope

- 기존 AWS Lookout for Vision 기반 운영을 RTM 학습/추론 솔루션으로 대체 또는 전환하는 흐름
- ROI 자동 지정
- 제품별 검사영역 설정
- PLC 연동
- 사내 추론서버 활용
- 신규 고정식 광학 시스템 도입

### Inspection Context

- 메일 발췌 기준으로 프레스 셔틀 `1/2라인`은 보일러 바디 생산, 단동 프레스 `35기`는 핀/캡 등 부품 생산에 사용된다.
- 검사 장비로 `비디오메타`, `3차원 측정기`, 하이트 게이지, 두께 게이지, 버니어캘리퍼스가 언급되며, 초품 검사 후 품질 부서가 회수해 정밀 측정하는 흐름이 적혀 있다.
- 검사 결과는 `엑셀`로 기록해 `One Drive`에 저장한다고 되어 있어, 후속 증거가 문서보다 스프레드시트 축에서 추가로 나올 가능성이 높다.
- 주요 불량 예시로 `찍힘`, `크랙`, `인각 깨짐`, `평탄 초과`, `버링 높이 미달`, `이물질`이 제시된다.

### Document Lineage

- `Pre-PoC`에서는 체결 로봇 동작시간 기반 이상탐지 가능성을 먼저 검토했다.
- `2차 제안서`에서는 비전 기반 검사, ROI 자동 지정, 구축안, 견적 구조까지 확장됐다.
- 후속 메일 발췌 문서에는 `1-1. (경동)연구개발계획서_본문1_v2_경동작성(진행).hwp`, `1-2. (경동)연구개발계획_본문2_v2_경동작성(진행).hwpx`가 직접 언급돼, 실제 국가과제형 연구개발계획서 작성 단계로 이어졌음이 확인된다.

### Budget

- 솔루션 라이선스 `25,000,000원`
- 개발 `112,100,000원`

- 라인 확장안 `150,000,000원`

### Risks

## 7. ZEUS_POC / BD 이물탐지

- 개별 페이지: [[RTM_YNG_ZEUS_POC]]

### Project Overview

- `ZEUS_POC 결과보고서`는 `CT(Coater)`와 `BD(Bonder)` NVR 영상 기반 편심·이물 감지 가능성을 검토한 기술 타당성 보고서다.
- 결론은 현재 촬영 조건에서는 이물 감지도, 편심 감지도 직접 도입 가능한 수준이 아니라는 쪽에 가깝다.
- 즉, 이 프로젝트는 성공 사례보다는 실패 조건과 다음 하드웨어 요구사항을 정리한 문서로 읽는 편이 맞다.

### Key Findings

- 이물은 전 구간에서 검출되지 않아 추적 자체가 불가능하다고 정리된다.
- 편심은 사선 촬영, 해상도, 고속 구간 흔적 소실 때문에 안정적 감지가 어렵다고 제시된다.
- 대안으로 수직 촬영, 해상도 상향, 레이저 프로브 등 하드웨어 보강이 제안된다.

### Risks

- 보고된 총 영상 수 `49개`와 세부 합계 `CT 20 + BD 19 = 39`가 맞지 않아 원문 재확인이 필요하다.
- `2024` 바우처 계열 `hwpx`는 이미 확보·열람했지만, `BD 이물/편심` 축과 `액적/스플래시` 축이 같은 제우스 프로젝트의 하위 실험인지까지는 아직 확정되지 않았다.

- 개별 페이지: [[RTM_YNG_반도체_공정제어_멀티에이전트]]

### Project Overview

- 과제명은 `소부장 온톨로지 연계 온디바이스 기반, 수율 극대화용 선제적 반도체 공정 제어 멀티에이전트 기술 개발`이다.
- 과제번호는 `RS-2026-25523818`, 연구개발기간은 `2026.04.01 ~ 2028.12.31`로 확인된다.
- 세정 공정을 시작점으로 온톨로지, 온디바이스 AI, 멀티에이전트, 피지컬 AI를 결합해 반도체 제조 자율운영 체계를 구축하려는 방향이 명확하다.

### Knowledge Points

- 1차 실증 범위는 세정 공정이며, 이후 식각·증착 등으로 확산하는 전략을 취한다.
- 단일 정확도보다 운영 안정성, 드리프트 대응, OTA, 감사 추적까지 함께 보는 도입형 지표 체계가 강조된다.
- `PSK`, `ZEUS`, 소재·부품사, `S2W`, `성균관대`가 연결되는 밸류체인형 실증 구조로 읽힌다.

### Risks

- 발표자료와 질의응답서 중심으로 읽은 상태라 전체 사업비와 세부 KPI는 아직 더 채워야 한다.
- `한양대` 후보군 `hwpx`와 이 과제가 직접 같은 트랙인지는 추가 확인이 필요하다.

### Budget / KPI

- 총과제비 `430,000천원`
- 정부지원금 `230,000천원`
- 민간부담금 `200,000천원`
- 물성치 추정 정확도 `0.9 이상`
- 설비 데이터 이상탐지 성능 `F1 score 0.9 이상`

### Risks

- `0318 -> 0321 -> 0322 -> 0323 -> 인건비 삭제본` 계보는 확인했지만, 사업비 세부 항목의 표 단위 diff는 아직 끝나지 않았다.
- `1.POC / 화승`의 영상/이미지 세트가 같은 과제의 실험 결과인지 별도 트랙인지 추가 확인이 필요하다.

## 10. 2024 AI바우처 / 제우스 / 픽셀

- 개별 페이지: [[RTM_YNG_제우스_픽셀_AI바우처]]

### Project Overview

- 현재 가장 강하게 읽히는 대표 축은 `제우스` 대상 `반도체 세정 장비 세정액의 비산 패턴 가상 계측 솔루션`이다.
- `TTA-24-1562` 시험자료 기준으로 내부 초고속 영상과 외부 비산 감수지 데이터를 연결해 비산 패턴을 계측·추정하는 구조가 명확하다.
- 같은 제출 준비 메모에서 `픽셀`도 별도 사업계획서와 성과자료 검토 대상으로 등장해, 운영상으로는 하나의 `2024 AI바우처` 취합 묶음으로 관리된 흔적이 있다.

### Knowledge Points

- 데이터 수집 기간은 `2024-06-11 ~ 2024-08-20`, 전체 공정은 `65회`, 프레임 총계는 `200,050`, 파일크기 총계는 `344GB`다.
- 핵심 목표는 비산 유형 객체인식 `F1 0.95 이상`과 외부 비산량 추정 정확도 `0.9 이상`이다.
- RTM의 `Apollo`와 `Hubble` 계열 모듈, `Virtual Metrology` 구조, 장비형/클라우드형 도입 방식이 함께 설명된다.
- 회사 공통정보 관점에서는 `제우스 = Hubble+Apollo 결합형 반도체 장비 가상계측 레퍼런스`, `픽셀 = Hubble 1.3 기반 PCB AOI 현장 성과 레퍼런스`로 분리해 쓰는 편이 안전하다.
- `제우스` 최종보고서에는 Hubble 객체 탐지/추적 `99.42%`, Apollo 공정 상태 예측 `100%`, 리드타임 `112시간 -> 35.2시간`, 약 `68.57%` 단축이 확인된다.
- `픽셀` 최종보고서에는 미검율 `0.15%`, 과검율 `0.91%`, `Takt time 1.354ms`, Hubble 객체 탐지 정확도 `99.9%`, Total Accuracy `99.962%`가 확인된다.

### Reuse Map

| 축 | 제안서/회사소개 재사용 문장 | 반드시 붙일 출처 |
|---|---|---|
| 제우스 | RTM은 Hubble 비전 객체인식과 Apollo 공정 예측을 결합해 반도체 세정 장비 비산 패턴 가상계측을 수행한 경험이 있다. | `zeus_final_sales.hwpx`, `[스크립트포함](AI반도체분과)_R-20240321-023009 _제우스_알티엠 .pptx` |
| 픽셀 | RTM의 Hubble 1.3은 PCB AOI 현장에서 미검/과검 저감과 짧은 Takt time 성과를 최종보고서 기준으로 남겼다. | `pixel_final.hwpx`, `3.발표자료_(주)픽셀.pptx` |
| 공통 | 두 과제 모두 브로셔성 claim이 아니라 AI바우처 결과보고서와 발표자료에서 직접 확인되는 고객 적용 증거다. | [[RTM_YNG_Company_Profile_2026-04-21]], [[RTM_YNG_Evidence_Log_2026-04-21]] |

### Risks

- `제우스`와 `픽셀` `hwpx` 최종보고서는 이미 확보·열람했지만, `매출기반`과 `매출기반X` 차이, 그리고 `TTA` 시험자료와 최종보고 수치의 정확한 대응 관계는 아직 더 봐야 한다.
- `제우스`와 `픽셀`이 같은 기술 과제인지, 같은 제출 주간 아래의 별도 과제인지 추가 판별이 필요하다.

## 11. 한양대 / 반도체 자율지능형 에이전트

- 개별 페이지: [[RTM_YNG_한양대_반도체_자율지능형_에이전트]]

### Project Overview

- 대표본은 `CY260116` 기반 `2026년 인공지능 기술사업화 지원사업 연구개발계획서_통합_0310.pdf`다.
- 과제는 반도체 장비 유지보수 현장의 정보 탐색과 의사결정을 지원하는 `On-Premise` 기반 산업 특화 `LLM` 에이전트 실증을 목표로 한다.
- `알티엠`, `서울대`, `한양대`, `PSK`가 함께 참여하며, 한양대는 온톨로지·지식그래프 축을 담당한다.

### Knowledge Points

- `10만건 이상` 도메인 문서, `Graph-aware RAG`, `Multi-Agent` 추론, `Fab` 실증이 핵심 키워드다.
- `라인 단위 On-Prem 공급 + 연간 갱신` 구조와 `5개년 누적 57억원` 사업화 계획이 직접 제시된다.
- 기존 `RS-2026-25523818` 공정제어 과제와는 별도 번호와 별도 사업 구조를 갖는 독립 트랙으로 보인다.

### Risks

- `기타업무` 쪽 조상 `hwp/hwpx` 계보는 파일명 기준으로 연결했지만, 각 중간본의 본문 diff와 수치 변화는 아직 정밀 비교하지 못했다.
- 한양대 폴더 내 세부 문서와 최종 제출본의 버전 계보는 초안 수준에서 정리됐고, 이제는 실제 내용 차이를 따라가야 한다.
