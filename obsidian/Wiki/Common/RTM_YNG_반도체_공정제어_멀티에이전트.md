# 반도체 공정제어 멀티에이전트

- [[Wiki/Common/RTM_YNG_반도체_공정제어_멀티에이전트_Status]]

## Project Overview

- 과제명은 `소부장 온톨로지 연계 온디바이스 기반, 수율 극대화용 선제적 반도체 공정 제어 멀티에이전트 기술 개발`이다.
- 과제번호는 `RS-2026-25523818`, 연구개발기간은 `2026.04.01 ~ 2028.12.31`로 확인된다.
- 핵심 방향은 반도체 세정 공정을 시작점으로, 온톨로지와 온디바이스 AI, 멀티에이전트, 피지컬 AI를 결합해 수율 극대화와 다운타임 최소화를 노리는 자율운영 체계를 만드는 것이다.

## Core Architecture

- `Ontology`: 공정 단계, 레시피, 장비 상태, 제약, SOP, 교체 이력까지 지식망으로 구조화
- `On-device`: 클라우드 의존을 줄이고 장비 인근에서 실시간 추론 수행
- `Agentic AI`: 감시, 추론, 제약 검증, 실행 추천을 오케스트레이션
- `Physical AI`: 실제 장비 제어 루프와 연결해 실행-피드백 구조를 지향

## Execution Plan

| 연차 | 실증 환경 | 중심 목표 |
| --- | --- | --- |
| 1차년도 | 연구소/실험실 테스트베드 | 세정 공정 기준 데이터 수집, 라벨링, 안정화 |
| 2차년도 | 파일럿 확장 테스트베드 | 장비/조건 확대, 부품·소재 이력 연계, 예지보전 모델 개발 |
| 3차년도 | 고객 FAB 연계 | 양산 환경 데모, 신뢰성 검증, 밸류체인 확장 |

## Value Chain

- 장비사 `PSK`, `ZEUS`와 소재·부품 기업이 함께 연결되는 구조를 전제한다.
- `S2W`는 온톨로지와 지식그래프 구축, `성균관대`는 예지보전 모델 개발과 평가 축으로 읽힌다.
- 과제는 단일 장비 지능화보다 `FAB-장비-부품-소재` 간 데이터 단절을 완화하는 밸류체인형 확장을 목표로 한다.

## Knowledge Points

- 1차 실증 범위는 웨이퍼 제조 전 공정 전체가 아니라 `세정(Cleaning)` 공정이다.
- 세정 공정 성공사례를 만든 뒤 식각, 증착 등으로 확산하는 템플릿 전략을 취한다.
- 기존 이상탐지 실증 경험을 바탕으로, 이상탐지에서 원인분석, 제어 추천, 자율 의사결정으로 단계적으로 확장하려는 설계가 명확하다.
- 반도체 환경 특성상 단일 정확도보다 운영 안정성, 드리프트 대응, OTA, 감사 추적 같은 도입 가능성 지표를 함께 본다.

## Risks

- 현재는 발표자료와 질의응답서 중심이라 전체 사업비와 세부 KPI 체계가 모두 닫혀 있지는 않다.
- `한양대` 축은 현재 [[RTM_YNG_한양대_반도체_자율지능형_에이전트]]의 `CY260116` 과제로 별도 분리되었다. 두 축 모두 반도체 에이전트 계열이지만 과제번호와 사업 구조가 다르다.
- 세정 공정 성공 후 식각/증착으로 확산한다는 전략은 명확하지만, 각 공정별 데이터 접근성과 검증 범위는 실제 실증 환경에 따라 크게 달라질 수 있다.
