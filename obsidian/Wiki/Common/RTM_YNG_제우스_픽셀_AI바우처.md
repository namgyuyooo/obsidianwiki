# 2024 AI바우처 / 제우스 / 픽셀

- [[Wiki/Common/RTM_YNG_제우스_픽셀_AI바우처_Reference_Register]]

## Project Overview

- 이 과제는 `2024 AI바우처` 계열에서 `제우스`를 중심 수요기업으로 두고, RTM이 반도체 세정 장비의 비산 패턴을 AI로 계측·추정하는 솔루션을 구축한 축으로 읽힌다.
- 대표 기술명은 `반도체 세정 장비 세정액의 비산 패턴 가상 계측 솔루션`이며, 내부 초고속 영상과 외부 비산 감수지 데이터를 연결해 공정 상태를 추정하는 구조다.
- `rclone + rhwp`로 `제우스 최종 hwpx`와 `픽셀 최종 hwpx`를 실제 열어보니, 두 축 모두 독립적인 본문·성과·특허·사업화 서술을 가진 최종보고 체계를 갖고 있었다.
- 같은 제출 준비 메모 안에 `제우스`와 `픽셀`이 함께 등장하므로, 운영상으로는 하나의 `2024 AI바우처` 제출 묶음 안에서 복수 과제를 취합한 흔적이 보인다. 다만 지금은 `제우스 주과제 + 픽셀 병행 기술과제` 구조로 보는 것이 가장 자연스럽다.

## Solution Structure

- 내부 비산 데이터:
  - `4,000fps` 초고속 카메라로 스핀컵 내부 비산 영상을 촬영하고, 프레임 단위 이미지로 변환한다.
- 외부 비산 데이터:
  - 스핀컵 외부 `60개` 비산 감수지의 흑점 면적을 정량화한다.
- AI 처리 구조:
  - 영상 객체인식으로 `droplet`, `droplet_splash`, `out`, `splash` 등 비산 유형을 분류한다.
  - 내부 비산 영상 결과와 외부 감수지 결과를 연결해 `Virtual Methodology` 기반 추정 모델을 구성한다.
- 제품 연결:
  - 시계열 기반 `Apollo`와 이미지 기반 `Hubble` 모듈이 함께 제시되며, 장비형/클라우드형 도입 구성이 설명된다.

## Pixel Workstream

- `픽셀` 쪽 별도 메모를 보면, 이 축은 `DMT` 고객사의 정상데이터 필터링 AI 프로젝트와 연결되어 있었고, `11월 내 테스트 환경 구축 + 약 2주간 실제 테스트 운영`이 목표였다.
- 협업 목적은 `픽셀-RTM` 연동 방안 확립, `DMT` 성능 만족도 확인, 후속 비즈니스 확장 기회 마련으로 적혀 있다.
- 기술적으로는 픽셀 `IVS`와 RTM 추론 모듈을 연결하는 구조가 논의됐고, RTM은 `ONNX`, 픽셀은 `Keras/TensorFlow 2.9.1` 기반 런타임을 사용 중인 것으로 적혀 있다.
- 메모에는 `RTM-inference.py`로 추론 브리지를 두고, 기존 `AI.XML`을 읽어 동일 양식의 판정 결과 XML을 생성하거나 `json` 등 합의된 방식으로 결과를 전달하는 안이 제안된다.
- `DMT` 현장 AI PC 사양은 `Intel Core i7-8700`, `RAM 48GB`, `NVIDIA GeForce 1080Ti`, `Windows 10 Pro`로 적혀 있어, 픽셀 축은 현장 추론 인프라까지 실제 검토되고 있었다.

## Risks

- `제우스`와 `픽셀`은 같은 제출 묶음에 있었지만, 기술 과제가 완전히 하나였는지 아니면 별도 과제를 같은 주간에 취합한 것인지는 추가 확인이 필요하다.
- `제우스` 최종본은 `매출기반 .hwpx`라는 실물 파일명으로 확보됐는데, 메모에서는 `매출기반O/X` 분기로 적혀 있어 파일명 정합성 확인이 더 필요하다.
- `344 GB`와 `200,050` 프레임 규모는 TTA 설명자료 기준으로 확인되지만, 최종 `hwpx` 본문에서 같은 숫자가 어떻게 재사용·정제됐는지는 아직 표 단위로 교차검증하지 못했다.
- `제우스 중간보고`와 `제우스 최종보고`의 성능 상승폭, 그리고 `매출기반X` 분기와의 차이는 아직 정밀 diff가 남아 있다.
