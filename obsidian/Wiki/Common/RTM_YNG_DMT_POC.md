# DMT_POC

- [[Wiki/Common/RTM_YNG_DMT_POC_Status]]

## Project Overview

- `DMT POC 결과보고서`는 불량 코드와 이미지 데이터를 바탕으로 외관 불량 탐지 모델을 구축하고, 데이터 정제와 필터링 로직까지 검토한 프로젝트다.
- 단순 모델 성능보다 `데이터 품질`, `불량 코드 매핑`, `과검/미검 제어`, `실운영 필터링률`이 핵심이다.

## Data Snapshot

| 항목 | 기준 |
| --- | --- |
| 전체 데이터 | `15,481장` |
| 불량 데이터 | `2,944장` |
| 정상 데이터 | `10,221장` |
| 기타 데이터 | `2,316장` |

## Modeling Notes

- 색상 분류 AI 모델을 통해 데이터를 `패턴(동)`, `도금`, `SR`, 기타 계열로 구분한 뒤 세부 영역 탐지 모델로 이어가는 구조다.
- 세부 불량 유형은 패턴/도금/SR 계열을 중심으로 총 `21개 클래스` 체계로 운영된다.
- 라벨링은 `2,817장` 이미지에 대해 약 `5,011개` 객체 수준으로 진행된 것으로 읽힌다.

## Result Snapshot

- 최종 정제 데이터 `8,074장` 기준 추론 진행
- 불량 데이터 정확도 `80.64%`
- 정상 데이터 정확도 `85.67%`
- 현재 필터링률은 `85%대`
- 정제 전 단계에서는 정상 데이터 중 실제 정상으로 추정 가능한 비율을 `66.44%` 수준으로 본다

## Core Issues

- 불량 코드가 없는 1차 데이터 `1,254장`이 존재해 코드 매핑이 필요하다.
- 전체 불량 코드 `120개` 중 실제 데이터가 있는 코드는 `30개`뿐이고, `90개`는 미수집 상태다.
- 허용영역 이물, 데이터 오염, 잘못 추론된 흑점, 과도한 추론이 주요 과검 요인으로 정리된다.

## Improvement Direction

- 임계값 조정
- 분류기 로직 고도화
- 색상분류기 고도화
- 추가 불량 코드 데이터 확보
- 코드 없는 1차 데이터의 코드 매핑
- 하이퍼파라미터 튜닝과 데이터 증강

## Risks

- 현재 성능의 병목은 모델 자체보다 데이터 정합성 문제일 가능성이 높다.
- 코드 누락, 데이터 오염, 허용영역 정의 문제를 정리하지 않으면 운영 단계에서 과검이 지속될 수 있다.

## Open Questions

- `90개` 미수집 불량 코드를 언제, 어떤 우선순위로 확보할 것인가
- 허용영역 이물은 운영 정책상 정상으로 볼지, 관리성 불량으로 볼지 정리됐는가
- DMT 측 의견과 RTM 측 의견이 다른 항목은 최종적으로 어떻게 정리되었는가
