# ZEUS_POC / BD 이물탐지

## Project Overview

- 대표본은 `ZEUS_POC 결과보고서`이며, `CT(Coater)`와 `BD(Bonder)` NVR 영상에서 편심과 이물 감지 가능성을 검토한 PoC 결과 문서다.
- 문서의 결론은 현재 촬영 조건에서는 이물 감지도, 편심 감지도 모두 바로 도입 가능한 수준이 아니라는 쪽에 가깝다.
- 즉, 이 프로젝트는 "성능이 좋다"는 성공 보고서보다, 어떤 조건에서 현 방식이 막히는지와 다음 하드웨어 조건이 무엇인지를 정리한 기술 타당성 보고서로 보는 편이 맞다.

## Findings

| 항목 | 결론 |
| --- | --- |
| 이물 감지 | 전 구간에서 이물이 검출되지 않아 추적 자체가 불가능 |
| 편심 감지 | 사선 촬영, 해상도, 고속 구간 흔적 소실 때문에 안정적 검출 불가 |
| 공정별 초점 | `BD`는 Idle time 이물 탐지, `CT`는 웨이퍼 회전 구간 편심 확인 목적 |

## Technical Notes

- `BD`는 `Wafer-out -> Initial -> Wafer-in(lower)` 구간을 중심으로 Idle time 이물 탐지를 시도했다.
- `CT`는 `process-start -> process-finished` 구간에서 실제 웨이퍼 회전 시 편심을 확인하려는 구조다.
- 문서상 핵심 한계는 현재 카메라의 설치 각도와 화각-거리 조건이 AI 모델 이전에 물리적으로 불리하다는 점이다.

## Alternative Direction

- 수직 촬영으로 카메라 각도를 재설계할 필요가 있다.
- 해상도를 높여 미세 흔적이 사라지지 않도록 해야 한다.
- 레이저 프로브나 전용 센서 같은 하드웨어 보강이 필요하다.

## Related Zeus Workstreams

- 별도 문서 `별첨. 240724)제우스 개발방안(2024.07.24)- RTM.pdf`에서는 웨이퍼 표면 액적/스플래시 분석과 `PIV` 유사 추적 가능성을 검토했다.
- 이 문서는 현재 `ZEUS_POC`와 직접 같은 문제를 다루지는 않지만, ZEUS 계열 과제가 단일 불량검사만이 아니라 고속 영상 기반 물리 현상 해석까지 확장되어 있음을 보여준다.
- 해당 개발방안 문서에서도 촬영환경 일관성 부족, 3D 왜곡 보정 한계, `TOF` 등 하드웨어 보강 필요성이 강조된다.

## Risks

- 보고서에 적힌 총 영상 수 `49개`와 세부 합계 `CT 20 + BD 19 = 39`가 일치하지 않아 원문 재확인이 필요하다.
- 현재 대표본은 "도입 가능"보다 "현 조건으로는 불가"에 가까운 결론이므로, 제안서형 과제와 같은 평가 기준으로 보면 안 된다.
- `2024` AI바우처 중간보고 `hwpx`를 아직 열지 못해 ZEUS 계열의 실제 사업 범위가 어디까지 이어지는지는 추가 확인이 필요하다.

## Open Questions

- 영상 총량 `49개`와 세부 수량 `39개`의 차이는 단순 오기인가, 누락된 분류가 있는가
- ZEUS 계열에서 `BD 이물/편심`과 `액적/스플래시`는 같은 프로젝트의 하위 실험인가, 별도 과제인가
- 후속 제안서나 하드웨어 재촬영 계획이 실제로 이어졌는가
