## Trust my Tech / 탈레스

- 계보 확인:
  - `별첨1-9. (탈레스) Trust my Tech 프로그램 사업계획서.pdf`
  - `(kr)별첨1-9. (탈레스) Trust my Tech 프로그램 사업계획서_vf.pdf`
  - `별첨1-11. Trust my Tech 프로그램 사업계획서(인건비 제외)_250509.pdf`
- 현재 판단:
  - 기본본은 양식/목차 흔적이 많이 남아 있는 템플릿 성격 문서로 보인다.
  - `vf`는 과제 주체, 제품명 `TS Agent`, 보안 요구, 글로벌 협업 확장 서술이 채워진 대표 최종 후보로 판단된다.
  - `인건비 제외본`은 단순 비용 삭제본이 아니라, 과제번호와 예산 구조, 솔루션 서술 축이 달라진 별도 변형본 성격이 강하다.
- 확인된 변화:
  - 기본본 -> `vf`
    - 템플릿/목차 중심 문서에서 실제 사업계획 내용이 채워진 제출형 문서로 완결성이 크게 상승했다.
    - `TS Agent`, 온프레미스, 접근통제, 접근 로그, 감사추적, 글로벌 제안 확대 같은 보안형 제조 현장 맥락이 강화됐다.
  - `vf` -> `인건비 제외본`
    - `vf` 기준 과제번호 `20446514`, 정부지원 `100백만원`, 총사업비 `143백만원` 구조에서
      `인건비 제외본`은 과제번호 `20286906`, 정부지원 `192,480,000원`, 총사업비 `274,980,000원`, 현금 `27,510,000원`, 현물 `54,990,000원` 구조로 바뀐다.
    - 제품/과제 서술도 `TS Agent` 중심 장비 유지보수형에서 `Hubble` 보안형 제조공정 최적화 solution 쪽으로 이동한 흔적이 있다.
- 후속 확인 필요:
  - `vf`와 `인건비 제외본`이 같은 공모의 비용 조정본인지, 다른 제안 트랙인지 확인
  - KPI 표와 사업화 항목이 어느 문서에서 처음 확정됐는지 확인

## AI바우처 / 케타포

- 계보 확인:
  - `AI바우처_사업계획서(26)_케타포_0.3.pdf`
  - `AI바우처_사업계획서(26)_케타포_0.5_한세민팀장님공유용.pdf`
  - `AI바우처_사업계획서(26)_케타포_최종취합.pdf`
- 현재 판단:
  - 버전 표기가 명확하고 `0.3 -> 0.5 -> 최종취합` 흐름이 확인되어 대표본 계보가 가장 깔끔하다.
  - `0.3`은 템플릿 흔적과 초안성이 강하고, `0.5`는 구조와 표현이 정리된 중간본, `최종취합`은 제출형 정리본으로 판단된다.
- 확인된 변화:
  - `0.3`
    - 앞부분에 양식/보일러플레이트 흔적이 남아 있고 표지/메타정보 완결성이 약하다.
    - 다만 후반부에는 이미 발주 예측, CS 자동화, 3D Bin-Packing, `71.6%` 자동응답 해결 같은 핵심 PoC 축이 들어가 있다.
  - `0.5`
    - `0.3` 대비 문서 구조와 front matter가 정리되고, 설명 흐름이 자연스러워졌다.
    - 핵심 과제 축 자체는 유지되어 프로젝트 방향 전환보다는 정리/보강 단계로 보인다.
  - `최종취합`
    - 주관/참여기관, 수행기간, 멀티에이전트 구조, KPI, 기대성과가 가장 완결적으로 제시된다.
    - 대표본으로 채택하기에 가장 안정적이다.
- 후속 확인 필요:
  - `0.3`, `0.5`, `최종취합` 간 기대성과 수치와 참여기관 표기 변화 여부
  - `71.6%` FAQ 자동화 수치와 최종 KPI 표의 연결 관계 확인

## 경동나비엔 / RTM

- 계보 확인:
  - `2025_경동나비엔_MCSC 과제 Pre-PoC결과 및 요구사항 논의_250205.pdf`
  - `250423_경동나비엔-RTM 과제 제안서 2차_04.pdf`
  - `250924_경동나비엔-RTM poc 제안서_00.pptx`
- 현재 판단:
  - `Pre-PoC` 단계에서는 체결 로봇 동작시간 데이터를 이용한 이상탐지 가능성과 LOB 분석 최적화가 먼저 논의됐다.
  - 이후 `2차 제안서`에서는 볼트/클립 누락 검사, 고정식 광학, ROI 자동지정, PLC 연동, 사내 추론서버, 1라인 견적까지 포함한 비전 기반 구축안으로 범위가 커졌다.
  - 다시 후속 `poc 제안서` 발표자료로 이어지는 고객 대응 흐름이 보인다.
- 확인된 변화:
  - `Pre-PoC` -> `2차 제안서`
    - 단일 feature 기반 이상탐지 검토(`IQR`, `Nelson Rule`)에서 현장 비전 검사와 시스템 구축 제안으로 초점이 이동했다.
    - 데이터 설명도 약 `6시간`, `15개 컬럼 x 29,585 raw`, `815개 제품`, `724개 바코드`, `SEQ 길이 37/35/6`처럼 탐색형 구조에서 검사 포인트/정확도/구축비 중심으로 바뀐다.
- 후속 확인 필요:
  - `txt/png` 부속 자료와 `poc 제안서`에서 검사 포인트 정의와 ROI 규칙이 어떻게 구체화됐는지 확인
  - 1라인 구축안과 라인 확장안의 가격/구성 변화

## HPP

- 계보 확인:
  - `[RTM] PoC 결과 보고HPP_final.pptx`
  - `[RTM]HPP_PoC 결과 보고서_최종.pdf`
  - `[RTM]HPP_PoC 결과 보고서_최종.pptx`
  - `HPP_final` 계열 문서
- 현재 판단:
  - `2025.11.27`자 `HPP_final`은 Doctor GAP 중심의 초기 기술검토/가정 문서에 가깝다.
  - 최종 배포용 PDF와 편집용 PPTX가 동시 존재하며, 큰 메시지는 거의 동일하다.
  - 다만 `pptx`는 배포본보다 추진 경과, 데이터 규모, 단계별 도입 절차를 더 상세히 담고 있다.
- 확인된 변화:
  - `HPP_final` -> 최종본
    - 초기본은 Doctor GAP 자동화 가능성, 테스트 영상 3건, `450`개 라벨링 데이터, 객체 검출과 ROI 체류시간 기반 판단 논리에 초점이 있다.
    - 최종본으로 가면서 E-ring, GAP, 출력물 검사, 외관 검사 4대 과제를 포괄하는 프로그램형 제안으로 확장된다.
    - ROI, Quick Win, 단계별 DX 로드맵, 후속 예산 약 `2.5억` 등 경영/확산 관점 서술이 크게 강화된다.
  - PDF <-> PPTX
    - 핵심 정량 성능과 후속 예산 약 `2.5억` 제안은 대체로 일치한다.
    - `pptx`에는 `2025.11.21` 초기 미팅, `2025.12.04` 현장 촬영, `2025.12.31` AI 파이프라인 개발, `2026.01.28` 2단계 완료 등 마일스톤이 더 또렷하다.
    - `pptx`에는 `정상 90개`, `불량 시료 15종`, 출력물 검사 `정상 21장 / 불량 18장`처럼 데이터 수량과 PoC 운영 조건이 더 많이 남아 있다.
- 후속 확인 필요:
  - `HPP_final` 외 다른 중간 계열 문서가 더 있는지 확인해 마일스톤과 데이터 수량이 언제 확정됐는지 추적
  - PDF 축약 과정에서 빠진 운영 리스크나 전제조건이 있는지 확인
