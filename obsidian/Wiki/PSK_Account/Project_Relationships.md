# Project Relationships

PSK 계정은 단일 프로젝트라기보다 여러 장기 workstream이 함께 움직이는 고객 계정이다. 현재 위키에서는 `PSK_Project` 폴더 하나 아래 주요 축을 하위 문서로 분리하는 구조가 가장 자연스럽다.

## 핵심 축

- `공동기술개발 계약 / 상업`
  계약 금액, 공수, 세금계산서, 서버 조달과 같은 상업 증적이 모이는 축이다.
- `EHM 운영`
  발주, 버전, export 이슈, 납기, 운영 환경이 모이는 축이다.
- `PE Agent`
  데모 운영, 설명자료, 서버 사양, 피드백 반영이 모이는 축이다.
- `Precia`
  milestone, 추가 데이터, threshold 일반화 한계가 모이는 축이다.
- `온도예측task`
  아직 초기 탐색 단계로, CTD팀 상견례와 IR 센서 요구사항 청취 중심의 하위 workstream이다.

## 위키 해석

- PSK는 `계약/납품/운영/제품화/현장적용`이 동시에 진행되는 장기 계정이다.
- 따라서 PSKH를 별도 독립 프로젝트로 분리하기보다, 현재는 PSK 상위 계정 아래에서 하위 문서로 다루는 편이 증적 흐름을 읽기 쉽다.
- 향후 PSKH 전용 계약, 납품, KPI가 독자적으로 더 쌓이면 별도 프로젝트 승격을 다시 검토할 수 있다.
