## 실제 운영 흐름 - 2026-04-30

1. 수집: 배포 변경 항목, result zip, raw/overlay 이미지, 실패 케이스, 공장/연구소 데이터 차이를 [[Evidence_Log]]에 보존한다.
2. 분석: 시각화 변경, crop 입력 구조, 해상도 이슈, 수식 기반 UA 계산, contour/vision 로직 고도화 후보를 분리한다.
3. 판단: 공장 데이터 기준 재학습이 필요한지, 현재 배포를 보정으로 유지할지 [[Decisions]]와 [[Risks]]에 올린다.
4. 실행: 실패 케이스 재현, 공장 이미지 확보, 항목별 정확도 확인, UI/overlay 개선을 [[Action_Items]]에 연결한다.
5. 보고: CEO는 납품/운영 경계, PM은 현장 데이터 기반 안정화 액션을 확인한다.
