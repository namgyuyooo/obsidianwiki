# Factory vs Lab Gap

금호타이어 CMS 프로젝트의 핵심 문제는 연구소 환경에서 보이던 품질이 공장 환경에서 그대로 유지되지 않는다는 점이다. 공개 Slack 기준으로 공장과 연구소가 중요하게 보는 항목이 다르고, 이미지 품질과 해상도도 달라 동일 로직을 그대로 적용하기 어렵다는 메시지가 반복된다.

## 핵심 차이

- 공장에서는 작업자가 중요하게 보는 8가지 항목 중심으로 먼저 맞추려는 요구가 있다.
- 연구소는 더 많은 항목을 보고 싶어 하지만, 실제 공정 적용 기준과는 겹치지 않는 부분이 있다.
- 공장 이미지의 해상도와 품질 차이 때문에 연구소 데이터로 학습한 모델이 현장에서는 충분히 안정적이지 않다.

## 대응 방식

- 실제 적용을 생각하면 공장 이미지를 기반으로 다시 학습하거나 개선하는 것이 더 현실적이라는 방향이 공유됐다.
- 일부 항목은 AI 검출 정확도가 부족해, 우선 수식 기반 계산으로 대응하고 추후 contour/vision 로직을 고도화하는 접근이 사용됐다.
- 이 과정에서 고객 내부 보고 일정이 1월 3주 데드라인으로 작동해, 원격 배포까지 고려한 빠른 반영이 필요했다.

## 위키 해석

- 금호 CMS는 단순 모델 성능 개선 프로젝트가 아니라, `연구소 기준`과 `현장 기준` 사이를 조정하는 현장 전환 프로젝트다.
- 다른 제조 비전 프로젝트에서도 `학습 데이터 기준`과 `실사용 데이터 기준`이 다를 때 어떤 trade-off가 생기는지 보여주는 전형적인 사례다.
