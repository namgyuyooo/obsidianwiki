# Deployment and Constraints

LG엔솔 이미지분석 프로젝트의 핵심은 모델 자체보다, 고객 데이터와 운영 환경 제약을 만족하는 배포 구조를 정하는 데 있다. 공개 Slack 기준으로 이 프로젝트는 계약 초안 검토와 동시에 `클라우드 사용`, `고객 이미지 반출`, `세션 유지`, `파일 삭제 정책`, `Task Queue`까지 함께 검토했다.

## 배포 방향

- 초기에는 `RTM 서버 혹은 AWS`가 모두 검토 대상이었다.
- 그러나 후속 메시지에서 `AWS 사용은 문제 없지만 엔솔 이미지가 올라가는 것은 불가`하다고 명시됐다.
- 그 결과 사내 개발 서버를 활용하는 방향으로 진행하겠다는 메시지가 확인된다.

## 운영 요구

- 이 프로젝트는 단순 추론 API가 아니라 서버 동시 사용자 수, 업로드 파일 규모, 세션 유지 방식, 업로드 파일 삭제 정책을 함께 설계해야 한다.
- Python 버전, Docker 지원, Message Queue 설치 가능 여부도 검토 항목으로 제시됐다.
- `ppt/pdf에서 이미지 추출 사용` 같은 편의 기능은 원본 데이터 품질과 운영 리스크 때문에 제외하는 방향으로 정리됐다.

## 위키 해석

- LG엔솔은 `이미지 분석 모델 프로젝트`이면서 동시에 `보안 제약이 강한 내부 배포 프로젝트`다.
- 향후 유사 고객에게도 `클라우드 가능 여부`와 `고객 데이터 반출 허용 여부`를 반드시 분리해 기록하는 템플릿으로 재사용할 수 있다.
