# Project Overview

현대모비스 프로젝트는 공개 Slack 기준으로 요구사항 문서, 제작사양서, 견적서가 연결된 제안-전환형 프로젝트다. 2026년 3월 말 요구사항 확인 문서가 생성되고, 4월에는 견적서 수정 요청과 납기/계약 일정 논의가 이어진다.

현재 핵심 구조는 `요구사항 확인 -> 제작사양 정리 -> 견적서 제출/수정 -> 계약 일정 협의` 흐름으로 보인다. 특히 메시지에서는 실무적으로 필요한 개발 기간과 계약서상 납기 요구 사이의 간극이 드러나므로, 일정 리스크 관리가 중요한 후보로 판단된다.

## 한 줄 요약

- 현대모비스 프로젝트는 요구사항과 제작사양은 구체화됐지만, 납기와 견적 방어 논리가 아직 조율 중인 제안-전환형 프로젝트다.

## 프로젝트 성격

- 유형: 요구사항 구체화 및 계약 전환 단계 프로젝트
- 고객: 현대모비스
- 주요 키워드:
  - ICT AI 솔루션 개발
  - 요구사항 확인
  - 제작사양
  - 견적서 수정
  - 납기 조정

## 현재까지 확인된 실제 정보

- 핵심 문서:
  - 요구사항 확인 문서
  - 제작사양서
  - 견적서
- 메시지에서 확인된 쟁점:
  - 본사 심의 대응을 위한 견적서 수정 필요
  - 내부 추정 개발 기간은 최소 3개월, 통상 4개월 수준으로 언급
  - 계약서 납기는 9월 말이 가장 현실적이고, 당기더라도 8월 말이 한계라는 판단이 공유됨
  - 프로젝트 시작 시점은 5월 초 전후로 예상됐으며, 4월 말 투자심의 이후 빠르게 kickoff할 가능성이 언급됨
  - 견적은 `99,000,000원 + alpha 20,000,000원` 수준 분리 검토와 `1+2 총액 1억원 이하` 기준이 함께 언급돼 상업 구조 조정 흔적이 있다
  - 서버 PC와 Risk Score 개발, EHM 라이선스가 견적 구성 요소로 언급됨
- 상태 해석:
  - 사업 제안은 진척됐지만, 일정과 견적 방어 논리가 아직 안정되지 않은 상태로 보인다.
  - 이 프로젝트의 핵심은 기능 정의보다 계약 가능한 일정/상업 조건을 만드는 데 있다.

## 현재 상태

- 요구사항은 고객/프로젝트 단위로 정리되어 있음
- 제작사양까지 문서화되어 구현 수준 논의가 시작됨
- 견적서는 수정 요청이 발생한 상태
- 일정은 계약상 납기와 실무 예상 기간이 충돌하는 상태
- 서버 사양과 데이터 처리량 조건도 일부 논의되어, 상업 제안이 이미 구현 전제와 연결된 상태다

## 위키에 남겨야 할 포인트

- 이 프로젝트는 `무엇을 만들까`보다 `언제까지 가능한가`가 핵심 리스크다.
- 요구사항 문서와 제작사양서를 같이 봐야 프로젝트 범위가 드러난다.
- 향후 수주형 프로젝트의 일정 리스크 사례로 재사용할 수 있다.
- 견적 방어는 단순 총액보다 `서버 + 알고리즘 + 라이선스` 구조를 어떻게 설명하느냐가 핵심이다.
