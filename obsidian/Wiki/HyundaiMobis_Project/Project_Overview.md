# Project Overview

현대모비스 프로젝트는 공개 Slack 기준으로 요구사항 문서, 제작사양서, 견적서가 연결된 제안-전환형 프로젝트다. 2026년 3월 말 요구사항 확인 문서가 생성되고, 4월에는 견적서 수정 요청과 납기/계약 일정 논의가 이어진다.

현재 핵심 구조는 `요구사항 확인 -> 제작사양 정리 -> 견적서 제출/수정 -> 계약 일정 협의` 흐름으로 보인다. 특히 메시지에서는 실무적으로 필요한 개발 기간과 계약서상 납기 요구 사이의 간극이 드러나므로, 일정 리스크 관리가 중요한 후보로 판단된다.

## 한 줄 요약

- 현대모비스 프로젝트는 요구사항과 제작사양은 구체화됐지만, 납기와 견적 방어가 아직 협의 중인 제안-전환형 프로젝트다.

## 프로젝트 성격

- 유형: 요구사항 구체화 및 계약 전환 단계 프로젝트
- 고객: 현대모비스
- 주요 키워드:
  - ICT AI 솔루션 개발
  - 요구사항 확인
  - 제작사양
  - 견적서 수정
  - 납기 조정

## 현재까지 확인된 실제 정보

- 핵심 문서:
  - 요구사항 확인 문서
  - 제작사양서
  - 견적서
- 메시지에서 확인된 쟁점:
  - 본사 심의 대응을 위한 견적서 수정 필요
  - 계약서상 납기 목표와 내부 수행 가능 기간 간 간극 존재
- 상태 해석:
  - 사업 제안은 진척됐지만, 일정과 견적 방어 논리가 아직 안정되지 않은 상태로 보인다.

## 핵심 엔티티

- 고객:
  - 현대모비스
- 문서:
  - 요구사항 확인
  - 제작사양서
  - 견적서
- 쟁점:
  - 심의 대응
  - 납기
  - 실무 공수

## 현재 상태

- 요구사항은 고객/프로젝트 단위로 정리되어 있음
- 제작사양까지 문서화되어 구현 수준 논의가 시작됨
- 견적서는 수정 요청이 발생한 상태
- 일정은 계약상 납기와 실무 예상 기간이 충돌하는 상태

## 위키에 남겨야 할 포인트

- 이 프로젝트는 `무엇을 만들까`보다 `언제까지 가능한가`가 핵심 리스크다.
- 요구사항 문서와 제작사양서를 같이 봐야 프로젝트 범위가 드러난다.
- 향후 수주형 프로젝트의 일정 리스크 사례로 재사용할 수 있다.
