# Project Overview

대덕전자 AFVI 프로젝트는 공개 Slack 기준으로 PCB/패키지 검사 영역에서 AFVI 제안과 기술 검토를 고도화해 온 프로젝트다. 이 채널에는 2023년 말부터 2025년 중반까지 제안서 버전, ODB 데이터, 인터페이스 논의, 크레셈 연동 검토, 진행 불안과 리스크 진단이 이어져 있어 단순 영업 후보가 아니라 실제 과제화 직전 또는 과제 수행 초입 단계의 프로젝트로 보인다.

특히 `AFVI 제안서`가 여러 버전으로 존재하고, 고객/협력사 측 요구에 따라 `ODB 데이터 소유권`, `I/O 정의`, `데이터 전송 방식`, `사용 Language`, `SDK 개발 담당`까지 논의된 점이 핵심이다. 이는 제안서 수준에서 이미 상세 업무와 기술 연동 방식을 구체화하려 했음을 의미한다.

## 한 줄 요약

- 대덕전자 AFVI 프로젝트는 AFVI 검사 제안과 ODB 기반 데이터/인터페이스 구조를 구체화한 제안-기술검토형 프로젝트다.

## 프로젝트 성격

- 유형: 제안-기술검토형 검사 프로젝트
- 고객: 대덕전자
- 주요 키워드:
  - AFVI
  - FCCSP
  - ODB
  - 크레셈 연동
  - SDK

## 현재까지 확인된 실제 정보

- 제안서 버전:
  - `01. 대덕전자 AFVI 제안서_231004.pptx`
  - `01. 대덕전자 AFVI 제안서_231128.pptx`
  - `231218_대덕전자 AFVI 제안서_ODB내용추가.pptx`
- 기술 자료:
  - `DD_afvi_도면여부_flow.pptx`
  - `I01_features`, `I06_features`, `S01_features`, `S06_features`
  - 네트워크 구성도
- 메시지 논의:
  - 기존 제안서는 상세 업무 내용, 개발 공수, 일정 산정이 부족하다는 내부 판단
  - ODB 데이터는 대덕전자 자산으로 보고, 필요한 범위를 받아 RTM이 이해/분석하는 방식 검토
  - 크레셈 연동을 위해 `SDK 개발 담당`, `I/O 정의`, `데이터 전송 방식`, `사용 Language` 확인 필요
  - 임원진 관심과 우려가 크며 AI 진행 상황이 잘 보이지 않는다는 메시지 확인

## 핵심 엔티티

- 협력/연동:

## 핵심 결정

- AFVI는 단순 모델 제안이 아니라 ODB 데이터와 설비/시스템 인터페이스 정의까지 포함하는 방향으로 정리됐다.
- 기존 제안서는 상세 업무와 공수/일정이 부족해 보완이 필요하다는 판단이 내려졌다.
- ODB 원본 데이터는 고객 자산으로 보고, 고객이 제공한 범위에서 RTM이 분석하는 방향이 채택됐다.

## 현재 상태

- 제안서와 기술 검토는 여러 차례 업데이트된 상태다.
- 하지만 채널 메시지에는 진행 상황이 고객/임원 관점에서 충분히 가시적이지 않다는 우려도 남아 있다.
- 따라서 현재 위키 분류는 “고도화된 제안 단계 또는 과제화 직전 단계”가 적절하다.

## 위키에 남겨야 할 포인트

- 이 프로젝트는 `제안서 버전 변화`와 `데이터/인터페이스 현실 조건`이 같이 드러나는 점이 중요하다.
- AFVI류 프로젝트에서는 데이터 소유권, ODB 구조, 인터페이스 요구가 제안 성공에 직접 연결된다는 사례로 재사용 가치가 높다.
- 기술 검토가 깊어질수록 오히려 진행 가시성이 떨어져 고객 불안이 커질 수 있다는 운영 리스크도 같이 남겨둘 만하다.

## 확인 필요

- AFVI 1차 제안과 이후 ODB 추가 버전이 하나의 과제 연속선인지
- 크레셈 연동 요구가 실제 수주 범위에 포함됐는지
- 내부 리스크 진단 이후 고객 보고 방식이나 일정이 어떻게 조정됐는지
