# Equipment (EPI Project)

## 2026-04-20 업데이트
- EPI(MOCVD) 공정 핵심 장비 정의와 데이터 특성 정리
- 장비 관점의 AS-IS 문제와 TO-BE 방향을 분리하여 구조화
- [[Equipment Health Manager]], [[Virtual Metrology]]와 연결되는 장비 설명 페이지로 재정리

## 1. 장비 개요
EPI(MOCVD) 공정에서 사용되는 핵심 장비와 운영 조건을 정리함.  
장비는 공정 품질(파장, 결함밀도, 수율)에 직접적인 영향을 미치므로 데이터 기반 관리가 필요함.

## 2. 주요 장비 리스트

### MOCVD 장비
- 장비 유형: GaN EPI 성장용 MOCVD
- 대표 장비: Veeco K465i 계열(예시)
- 역할: 웨이퍼 위에 GaN 결정층을 성장시키는 핵심 설비
- 운전 특성:
  - Run 단위 공정 운영
  - 1 Run 기준 약 3~6시간 소요
  - 레시피 기준 50~100 Step 내외로 구성
  - 센서/상태 채널 200~300개 이상 발생
  - 고온 단일 공정 특성상 공정 내부 상태의 직접 관측이 어려움

### 공정 관련 센서 및 상태값
- 온도
- 압력
- 가스 유량
- RF 파워
- 설비 상태값
- 이벤트 및 알람

### 장비 데이터 관점 요약
- 다변량 시계열 데이터가 1초 이하 단위로 발생
- 동일 Run 내부에서도 Step별 패턴이 상이함
- 장비 조건, 레시피, 이벤트를 함께 보지 않으면 원인 해석이 왜곡될 수 있음

## 3. 데이터 특성

- 데이터 유형
  - 시계열 데이터: Sensor, Trace
  - 이벤트 데이터: Alarm, State, Event Log
  - 레시피 메타데이터: Step, 조건값, Run 정보
  - 품질 데이터: Peak Wavelength, 기타 품질 계측값

- 데이터 구조
  - Run 단위 집계 구조
  - Step 기반 분할 구조
  - 장비별/라인별 분산 저장 구조
  - 설비 데이터와 품질 데이터의 시차 존재 가능

- 분석 시 유의점
  - 동일 알람이라도 장비 상태와 Step 맥락에 따라 의미가 달라질 수 있음
  - 품질 결과는 공정 종료 후 지연 확인되는 경우가 많아 실시간 라벨 확보가 어려움
  - 단일 센서 이상보다 복합 변수 패턴 변화로 해석해야 하는 경우가 많음

## 4. 현재 문제

- 장비별 데이터 파편화로 인해 통합 분석이 어려움
- 공정 내부 상태를 직접 관측하기 어려워 원인 규명이 지연됨
- 품질 결과 확인에 24~48시간 수준의 지연이 발생함
- 장비 간 품질 편차가 커 동일 레시피라도 결과 일관성이 낮을 수 있음
- 엔지니어 경험 의존 분석 비중이 높아 대응 표준화에 한계가 있음
- 설비 데이터, 이벤트, 품질 정보를 하나의 Run 맥락으로 연결해 보기 어려움

## 5. 개선 방향 (TO-BE)

- 실시간 데이터 수집 및 통합 구조 확보
- Run 단위 데이터 정합 체계 구축
- 이상탐지, 품질예측, 레시피 추천의 연계 분석 구조 마련
- 장비 상태를 설명하는 Health Score 체계 정의
- [[Virtual Metrology]] 기반 품질 예측 체계 고도화
- [[Equipment Health Manager]]와 연결되는 운영형 분석 체계 확립

## 6. 확인 필요
- 실제 적용 장비 수와 장비별 채널 수 확정 필요
- 대표 장비명이 Veeco K465i로 고정되는지 확인 필요
- 품질 편차 수치와 확인 지연 시간은 실증 문서 기준으로 재검증 필요
- 이벤트 데이터와 품질 데이터의 실제 연결 방식 정리 필요
