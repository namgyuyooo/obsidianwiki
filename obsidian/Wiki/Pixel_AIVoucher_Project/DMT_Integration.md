# DMT Integration

DMT 채널은 현재 기준으로 픽셀과 별개의 독립 고객 프로젝트라기보다, 픽셀 후속 구현과 현장 검증을 구체화하는 운영 보조 workstream에 가깝다. 공개 Slack 기준으로 이 채널에서는 `XML 구조 해석`, `_DL.xml` 인식, `NAS 경로`, `CSV 결과 제공`, `원격 접속 환경`, `테스트 PC 사양` 같은 실제 운영 항목이 상세하게 정리된다.

## 현재까지 확인된 실제 정보

- 2025년 1월 기준으로 `AI_PC에서 우선 동작 확인`, `_DL.xml` 인식 가능 IVS 버전 확인, `NAS 동일 위치 업로드`가 운영 절차로 정리됐다.
- 추론 대상은 NAS의 특정 LOT 폴더를 읽어와 이미지와 XML을 함께 복사하고, 결과를 `(원본파일명)_DL.xml`로 생성해 다시 업로드하는 흐름으로 설명된다.
- 2025년 7월 기준 테스트 계획에서는 `RTX 4090`, `Memory 128GB`, `2TB NVMe SSD`를 사용하는 테스트 PC와 `일 단위 CSV 결과 공유` 구조가 제시됐다.

## 위키 해석

- DMT는 픽셀 프로젝트의 `운영 반영/검증`을 맡는 구현 축이다.
- 따라서 독립 프로젝트 폴더로 승격하기보다, 픽셀 문서 안에서 `통합/배포 workstream`으로 관리하는 편이 자연스럽다.
- 이 문서는 향후 패키지 검사 프로젝트에서 `AI 결과를 기존 IVS/XML/NAS 흐름에 붙이는 방식`의 템플릿으로도 재사용할 수 있다.

## 확인 필요

- DMT 테스트 결과가 실제 계약 또는 정식 운영 전환으로 이어졌는지
- CSV 결과 형식이 최종 운영 포맷으로 확정됐는지
- DMT와 Pixel의 역할 경계가 고객 관점에서도 동일하게 설명되는지
