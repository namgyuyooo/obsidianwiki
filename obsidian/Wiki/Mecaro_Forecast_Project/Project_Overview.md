## 기존 정리 메모
메카로 수요예측 프로젝트는 공개 Slack 기준으로 `Pre-PoC 리뷰 -> PoC 결과발표 -> 단계별 제안서 -> 근거자료 질의 -> 생산/설비팀 미팅` 흐름이 비교적 선명한 제안-전환형 프로젝트다. 파일 구조만 보아도 내부 검토와 고객 제안, 근거자료 보강이 순차적으로 이어진 것으로 해석된다.

이 프로젝트의 강점은 제안 문서와 결과발표가 모두 존재한다는 점이고, 핵심 리스크는 제안 논리와 실제 FCST 근거자료 정합성에 있다. 즉, 수요예측 모델의 성능보다는 제안 구조와 데이터 근거 방어가 중요하게 보인다.

## 한 줄 요약

- 메카로 수요예측 프로젝트는 PoC 결과를 바탕으로 단계별 FCST 제안을 만들고, 그 제안의 근거자료를 보강해가는 프로젝트다.

## 프로젝트 성격

- 유형: 제안-전환형 프로젝트
- 고객: 메카로
- 주요 키워드:
  - 수요예측
  - FCST
  - 판매계획 데이터 자산화
  - PoC 결과발표
  - 근거자료 문의

## 현재까지 확인된 실제 정보

- 흐름:
  - Pre-PoC 결과 리뷰
  - PoC 결과발표
  - 단계별 프로젝트 수행 제안
  - FCST 작성 근거자료 문의
  - 생산/설비팀 미팅
- 메시지에서 확인된 추가 맥락:
  - 이 프로젝트는 IT 단독 검토가 아니라 영업 측 TF와 데이터 자산화 과제가 함께 얽힌 구조로 보인다
  - 1월 27일 기준 생산/설비팀과의 미팅이 별도 잡혀 실제 운영 부서 연결이 시작됐다
  - `메카로_FCST_현황` 자료가 공유되어 제안 이후에도 수요예측 현황을 계속 업데이트한 흐름이 있다
- 상태 해석:
  - 단순 PoC에서 끝난 것이 아니라, 결과를 사업 제안 구조로 번역하는 단계까지 진행됐다.
  - 따라서 메카로는 단일 모델 검증보다 `PoC 결과를 사업 언어로 번역하는 프로젝트`에 가깝다.

## 현재 상태

- 제안 문서와 결과발표가 모두 존재한다
- 근거자료 문의가 별도로 있는 것으로 보아 제안 방어 논리는 아직 보강 중이다
- 생산/설비팀 미팅 단계까지 이어져 현업 적용 검토가 시작된 상태로 읽힌다
- 수요예측 자체보다 판매계획 데이터 자산화와 사업 제안의 연결이 중요한 상태다

## 위키에 남겨야 할 포인트

- 이 프로젝트는 모델 성능보다 `제안 구조`와 `근거자료 정합성`이 더 중요한 사례다.
- PoC 결과발표와 단계별 제안서를 같이 봐야 사업화 논리가 보인다.
- 향후 FCST/수요예측 계열 프로젝트의 템플릿 사례로 재사용 가능하다.
- 영업 TF와 생산/설비 현업이 함께 등장한다는 점에서 내부 사업화 프로세스 사례로도 쓸 수 있다.

- 근거자료 문의에 대한 후속 답변과 실제 반영 여부
