# Project Relationships

서울바이오시스 계정은 `EHM 데모 POC`, `AI agent 방향 확장`, `상업 전환 논의`가 순차적으로 연결되는 구조다. 현재 위키에서는 하나의 `SeoulBiosys_Project` 아래에서 기술 검증과 상업 전환을 각각 하위 문서로 설명하는 방식이 가장 자연스럽다.

## 핵심 축

- `POC 축`
  EHM 소개, 데모 결과 공유, 2차 결과 보고가 이 축의 중심이다.
- `확장 축`
  AI agent 통합 인사이트처럼 POC 이후 더 넓은 방향으로 확장되는 논의가 이 축에 해당한다.
- `상업 전환 축`
  가격, 기존 FDC 대비 설명, 의사결정 지연, next step이 이 축의 중심이다.

## 위키 해석

- 서울바이오시스는 단순 데모 고객이 아니라, POC 성공 이후 실제 도입 전환의 난점을 보여주는 계정이다.
- 따라서 `POC_to_Agent`와 `Commercial_and_Next_Step`을 함께 읽어야 전체 흐름이 보인다.
- 향후 추가 계약/납품 증적이 생기면 하위 프로젝트 분리보다 현재 계정 안에 `도입 단계` 문서를 더하는 편이 먼저일 가능성이 높다.
